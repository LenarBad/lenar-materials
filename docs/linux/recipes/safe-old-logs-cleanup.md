# Безопасная чистка старых логов

## Индекс

- [Рецепты Linux](README.md)

## Когда использовать

Каталог логов приложения разрастается, а удалять нужно только **устаревшие** файлы (например старше 14 дней), не трогая текущие. Нельзя полагаться на «на глаз» или на `rm -rf` по широкой маске без предпросмотра.

## Пошаговый подход

1. **Preview** — выполнить `find` с теми же фильтрами (`-type`, `-name`, `-mtime`), что планируете для удаления, и убедиться, что в вывод попадают только нужные пути.
2. **Список** — записать тот же вывод в файл и просмотреть его (при необходимости отредактировать список вручную).
3. **Удаление** — применить `rm` только по сохранённому списку (например через `xargs`), а не повторный `find | xargs rm` с другими условиями.
4. **Проверка** — оценить занимаемый объём каталога и при необходимости общее место на томе (`df`).

Пути и порог `-mtime +N` подставьте под свою систему; ниже `/var/log/myapp` и `*.log` — пример.

## Команды

Предпросмотр кандидатов на удаление:

```bash
find /var/log/myapp -type f -name "*.log" -mtime +14 -print
```

Сохранить тот же список в файл:

```bash
find /var/log/myapp -type f -name "*.log" -mtime +14 -print > /tmp/myapp-old-logs.txt
```

Удалить только файлы из списка:

```bash
xargs -a /tmp/myapp-old-logs.txt rm -f
```

Проверить результат по каталогу:

```bash
du -sh /var/log/myapp
```

При сомнениях в свободном месте на разделе:

```bash
df -h
```

## Проверка результата

Размер каталога логов снизился ожидаемо; свежие логи на месте; сервис приложения при необходимости перезапуска не требует (если вы не удаляли открытые активные файлы — в типичной схеме ротации это учтено).
