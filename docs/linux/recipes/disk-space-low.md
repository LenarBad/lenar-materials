# Нехватка места на диске

## Индекс

- [Рецепты Linux](README.md)

## Когда использовать

Сообщения `No space left on device`, рост `/var/log`, алерты по `df`.

## Пошаговый подход

1. Понять, какой том и заполнены ли inode.
2. Найти каталоги и файлы, которые занимают больше всего места.
3. Освободить место осознанно (ротация, архив, удаление только проверенных артефактов).
4. Повторно проверить `df` и работу сервисов, которым нужен диск.

## Команды

```bash
df -h
```

```bash
df -i
```

Оценка объёма по ветке (путь при необходимости сузьте, например до `/var/log`):

```bash
sudo du -xh /var --max-depth=2 2>/dev/null | sort -h | tail -n 30
```

После удаления или ротации файлов снова проверьте свободное место:

```bash
df -h
```

## Проверка результата

Свободное место и inode в норме; критичные сервисы в `active`.
