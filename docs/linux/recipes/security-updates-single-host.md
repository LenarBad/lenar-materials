# Обновления безопасности (один хост)

## Индекс

- [Рецепты Linux](README.md)

## Когда использовать

Регулярное обслуживание или после CVE; согласовать окно простоя с политикой команды.

## Пошаговый подход

1. Обновить индекс пакетов и оценить список обновлений.
2. Установить обновления по политике команды (все или выборочно).
3. При обновлении ядра или критичных библиотек запланировать перезагрузку, если она требуется.
4. Проверить, что менеджер пакетов не оставил незавершённых операций и что сервисы подняты.

## Команды

### Debian/Ubuntu

```bash
sudo apt update
```

```bash
apt list --upgradable
```

```bash
sudo apt upgrade
```

### RHEL (dnf: Fedora, AlmaLinux, Rocky и др.)

```bash
sudo dnf check-update
```

```bash
sudo dnf upgrade
```

## Проверка результата

Нет застрявших или незавершённых установок пакетов; после ребута (если был) сервисы подняты (`systemctl` / смоук-тесты).
