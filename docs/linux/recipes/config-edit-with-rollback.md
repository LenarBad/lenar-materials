# Правка конфига с возможностью отката

## Индекс

- [Рецепты Linux](README.md)

## Когда использовать

Любые изменения в `/etc`, влияющие на запуск сервиса.

## Пошаговый подход

1. Сделать резервную копию файла до правок.
2. Изменить конфиг; для `sudoers` использовать только `visudo`.
3. Если у сервиса есть проверка конфигурации — выполнить её перед перезагрузкой процесса.
4. Применить изменения через `reload` или осторожный `restart`, держа под рукой путь к копии для отката.

## Команды

Резервная копия перед правкой (пути замените на свои):

```bash
sudo cp -a /etc/myapp/config.yml /etc/myapp/config.yml.bak.$(date +%Y%m%d)
```

Правка sudoers только через:

```bash
sudo visudo
```

Проверка синтаксиса nginx перед применением:

```bash
sudo nginx -t
```

Применить конфиг без обрыва соединений там, где поддерживается reload:

```bash
sudo systemctl reload nginx
```

## Проверка результата

Сервис активен; логи без новой ошибки парсинга конфига.
