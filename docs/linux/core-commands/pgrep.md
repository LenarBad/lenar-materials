# pgrep

## Назначение

**Найти PID процессов** по имени или шаблону командной строки без ручного разбора вывода `ps`. Удобно перед `kill`, проверкой аргументов сервиса или поиском «зависших» воркеров.

## TLDR

Найти PID по имени процесса.

```bash
pgrep nginx
```

Показать PID и полную командную строку.

```bash
pgrep -af nginx
```

Искать по полной командной строке (регулярное выражение).

```bash
pgrep -af "python.*worker"
```

Только количество совпадений.

```bash
pgrep -c sshd
```

Ограничить список конкретным пользователем.

```bash
pgrep -u www-data -a php-fpm
```

## Синтаксис

```text
pgrep [опции] шаблон
```

**Шаблон** по умолчанию сопоставляется с именем процесса; с **`-f`** — со всей командной строкой.

## Поведение и параметры

Команда выводит **один PID на строку** (если не заданы режимы счётчика). Имя процесса — это обычно **базовое имя исполняемого файла**, а не полный путь.

| Параметр | Что делает |
| --- | --- |
| `-a`, `--list-full` | Показать полную командную строку |
| `-f`, `--full` | Искать по всей командной строке, не только по имени |
| `-u`, `--euid` | Фильтр по эффективному пользователю (UID или имя) |
| `-c`, `--count` | Вывести только число совпадений |
| `-n` / `-o` | Выбрать только самый новый/старый из совпавших (зависит от версии; см. `man pgrep`) |

Путаница с **`-f`**: без него шаблон `python` может не совпасть с `python3.11`. С **`-f`** совпадение идёт по всей строке — уточняйте в `man pgrep`.

## Примеры

Убедиться, что нашли именно тот процесс, прежде чем слать сигнал.

```bash
pgrep -af "myapp"
ps -fp "$(pgrep -f '^/opt/myapp/bin/server' | head -n1)"
```
