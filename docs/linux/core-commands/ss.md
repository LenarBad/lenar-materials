# ss

## Назначение

Показать **сокеты**: соединения TCP/UDP, слушающие порты, связанные процессы. Быстрая альтернатива `netstat` на системах с iproute2.

## TLDR

Все TCP/UDP listening и установленные сокеты с процессами и числовыми адресами.

```bash
ss -tulpen
```

Только слушающие TCP с PID (часто нужен root для всех процессов).

```bash
ss -ltnp
```

Краткий вид.

```bash
ss -s
```

Фильтр по порту назначения.

```bash
ss -tlp 'sport = :443'
```

Сокеты для конкретного процесса.

```bash
ss -tp 'pid = 1234'
```

## Синтаксис

```text
ss [опции] [фильтр]
```

**Фильтры** позволяют сузить вывод по портам, адресам, состояниям (например `state listening`).

## Поведение и параметры

Без привилегий колонка **Process** может быть пустой для чужих сокетов — запускайте с **`sudo`**, если нужны PID всех служб.

| Параметр | Что делает |
| --- | --- |
| `-t` | TCP |
| `-u` | UDP |
| `-l` | Только listening |
| `-a` | Все состояния |
| `-n` | Числовой формат (без обратного DNS) |
| `-p` | Показать процесс |
| `-e` | Дополнительная информация о сокете |
| `-o` | Таймеры (keepalive и т.п.) |

Состояния TCP вроде **ESTAB**, **TIME-WAIT** помогают понять, висят ли старые соединения или исчерпан пул портов.

## Примеры

Проверить, кто слушает порт 5432 локально.

```bash
sudo ss -tlnp 'sport = :5432'
```
