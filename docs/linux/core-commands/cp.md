# cp

## Назначение

**Скопировать** файлы и каталоги: снимок данных в другом месте, бэкап перед правками, развёртывание файлов из шаблона. Базовая операция рядом с `mv` и `rm`; для повторяющейся синхронизации деревьев чаще смотрят на `rsync`.

## TLDR

Один файл в другое имя или каталог.

```bash
cp /etc/hosts /tmp/hosts.bak
```

Сохранить атрибуты и рекурсивно скопировать дерево (архивный режим GNU).

```bash
cp -a /srv/app /var/backups/app-snapshot
```

Рекурсивно без «архива» (поведение прав и ссылок уточняйте в `man cp`).

```bash
cp -r /opt/vendor/docs /usr/local/share/
```

Интерактивно спросить перед перезаписью.

```bash
cp -i ~/.bashrc ~/.bashrc.bak
```

Не перезаписывать существующий файл.

```bash
cp -n draft.txt final.txt
```

Подробный вывод: что куда скопировалось.

```bash
cp -av /etc/nginx/sites-available/default /tmp/
```

Сохранить только указанные метаданные (GNU).

```bash
cp --preserve=mode,timestamps report.txt /archive/
```

## Синтаксис

```text
cp [опции] источник ... назначение
```

Несколько **источников** допустимы, если **назначение** — каталог. **Завершающий слэш** у пути к каталогу влияет на то, копируется ли сам каталог или только содержимое — проверяйте по `man cp` на своей системе.

## Поведение и параметры

По умолчанию `cp` **перезаписывает** существующий файл с тем же именем без вопросов (в интерактивной оболочке удобнее **`-i`**). Права и владелец у копии зависят от режима и пользователя (часто копия принадлежит тому, кто запустил команду).

| Параметр | Что делает |
| --- | --- |
| `-a`, `--archive` | Рекурсивно + максимально сохранить атрибуты (GNU; детали в man) |
| `-r`, `-R` | Рекурсивно для каталогов |
| `-i`, `--interactive` | Спрашивать перед перезаписью |
| `-n`, `--no-clobber` | Не перезаписывать существующий файл |
| `-u`, `--update` | Копировать только если источник новее или файл отсутствует |
| `-v`, `--verbose` | Печатать имена |
| `-p` | Сохранять владельца/время/права (точный набор — в man; на GNU часто близко к «preserve metadata») |
| `--preserve[=список]` | Явно выбрать, что сохранять |

Копирование между **файловыми системами** — это фактически чтение и запись: жёсткие ссылки на исходе не дублируются как те же inode. Для **символических ссылок** поведение задаётся опциями (например `--dereference` / `-L` vs `-P` — см. `man cp`).

## Примеры

Копия конфига перед редактированием.

```bash
cp -i /etc/myapp/config.yml /etc/myapp/config.yml.bak
```
