# dmesg

## Назначение

Показать **сообщения ядра**: загрузка драйверов, ошибки устройств, **OOM-killer**, сетевые и дисковые предупреждения. Дополняет `journalctl` при разборе проблем на уровне железа и драйверов.

## TLDR

Последние строки буфера ядра.

```bash
dmesg | tail -n 100
```

Человекочитаемые временные метки.

```bash
dmesg -T | tail -n 50
```

Только ошибки и предупреждения (уровни зависят от версии).

```bash
dmesg -T -l err,warn
```

Следить за новыми сообщениями.

```bash
dmesg -w
```

## Синтаксис

```text
dmesg [опции]
```

На части систем доступ ограничен: может потребоваться **`sudo`** или настройка **sysctl** `kernel.dmesg_restrict`.

## Поведение и параметры

Буфер **ограничен по размеру**; старые сообщения могут исчезнуть. Для долгой истории смотрите также **`journalctl -k`**.

| Параметр | Что делает |
| --- | --- |
| `-T`, `--ctime` | Время в виде даты/времени вместо секунд от boot |
| `-l`, `--level` | Фильтр по уровням (`err`, `warn`, `info` и др.) |
| `-f`, `--follow` / `-w` | Поток новых сообщений (где поддерживается) |
| `-H` | Просмотр в pager с удобной подсветкой (util-linux) |
| `--since`, `--until` | Интервал (не на всех сборках; иначе pipe в `journalctl`) |

Сообщения вроде **I/O error**, **Out of memory**, **blocked for more than** — повод копать глубже по стеку железа/ФС/нагрузки.

## Примеры

Поиск упоминаний OOM в последних строках.

```bash
dmesg -T | rg -i 'out of memory|oom|killed process' | tail
```
