# rsync

## Назначение

**Синхронизировать** каталоги и файлы между локальными путями или по сети: копирование с контролем изменений, типичный сценарий — бэкапы, выкладка сайта, зеркалирование логов. Часто предпочтительнее «голого» `cp` при повторных переносах и при необходимости **dry-run**.

## TLDR

Архивный режим: права, время, рекурсивно (как правило — базовый выбор).

```bash
rsync -a src/ dst/
```

Предпросмотр: что изменится, без записи на диск.

```bash
rsync -a --delete --dry-run src/ dst/
```

Применить синхронизацию: удалить в `dst/` то, чего нет в `src/`.

```bash
rsync -a --delete src/ dst/
```

Показать прогресс и сохранить частично скопированные файлы при обрыве.

```bash
rsync -aP src/ user@host:/var/backups/project/
```

Исключить каталоги из копирования.

```bash
rsync -a --exclude '.git' --exclude 'node_modules' ./ project/ backup/
```

По SSH на удалённый хост (путь после `:`).

```bash
rsync -a -e ssh ./data/ user@backup.example:/srv/mirror/data/
```

## Синтаксис

```text
rsync [опции] источник ... назначение
```

**Завершающий слэш** у источника важен: `src/` — содержимое каталога, `src` — сам каталог как элемент.

## Поведение и параметры

`rsync` сравнивает списки файлов и при необходимости передаёт только отличия (в зависимости от режима и версии). Ключ **`-a`** (`--archive`) на практике задаёт рекурсию и сохранение типичных метаданных; точный набор флагов смотрите в `man rsync` на своей системе.

| Параметр | Что делает |
| --- | --- |
| `-a`, `--archive` | Архивный режим (рекурсия + типичные атрибуты; детали — в man) |
| `-v` | Подробный список передаваемых имён |
| `-n`, `--dry-run` | Только показать действия, не менять файлы |
| `--delete` | Удалить в назначении файлы, которых нет в источнике |
| `-P` | Часто `-P` = `--partial --progress` (удобно для больших файлов) |
| `--exclude=PATTERN` | Не копировать пути, подходящие под шаблон |
| `-e ssh` | Задать оболочку передачи (часто SSH для удалённой стороны) |

Перед первым боевым запуском с **`--delete`** обязательно прогоните с **`--dry-run`**. Ошибка в путях может стереть не тот каталог.

## Примеры

Локальное зеркало с удалением лишнего только после проверки.

```bash
rsync -avn --delete /srv/app/current/ /srv/app/staging/
rsync -av --delete /srv/app/current/ /srv/app/staging/
```
