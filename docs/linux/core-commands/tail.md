# tail

## Назначение

Показать **конец файла** (последние строки): быстро посмотреть хвост лога, свежие записи после ошибки. Режим **`-f`** непрерывно выводит новые строки — удобно рядом с **`journalctl -f`**, когда смотрите обычный текстовый файл.

## TLDR

Последние 10 строк файла (поведение по умолчанию часто 10 строк).

```bash
tail /var/log/nginx/access.log
```

Последние N строк.

```bash
tail -n 100 /var/log/syslog
```

Следить за ростом файла в реальном времени (выход — Ctrl+C).

```bash
tail -f /var/log/myapp/app.log
```

Переподключаться к файлу после ротации логов (имя пересоздали).

```bash
tail -F /var/log/myapp/app.log
```

Несколько файлов с заголовками.

```bash
tail -n 5 /var/log/syslog /var/log/auth.log
```

С момента байта в файле (редко; для больших файлов).

```bash
tail -c 4096 /var/log/big.log
```

## Синтаксис

```text
tail [опции] [файл ...]
```

Без **файла** часто читается стандартный ввод.

## Поведение и параметры

**`-f`** удерживает открытый файл и печатает новые данные; если файл **переименовали** (ротация логов), без **`-F`** можно «застрять» на старом inode — **`-F`** как раз помогает переподключаться по имени (уточните детали в `man tail` на своей системе).

| Параметр | Что делает |
| --- | --- |
| `-n`, `--lines=N` | Последние N строк (вместо дефолта) |
| `-f`, `--follow` | Ждать новые строки в конце файла |
| `-F` | Как `-f`, с переподключением при смене файла (типично для ротации) |
| `-c`, `--bytes=N` | Последние N байт |

Для структурированных журналов systemd удобнее **`journalctl -u … -f`**, чем сырой файл.

## Примеры

Последние ошибки в логе приложения после деплоя.

```bash
tail -n 200 /var/log/myapp/app.log | grep -i error
```
