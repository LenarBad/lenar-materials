# chown

## Назначение

Изменить **владельца** и/или **группу** файла или каталога. Нужно после копирования данных от root, при переносе приложения между учётными записями и при устранении ошибок вида «Permission denied» из‑за неверного владельца.

## TLDR

Владелец и группа сразу.

```bash
sudo chown app:app /var/lib/myapp/data.db
```

Только владелец; группа как была.

```bash
sudo chown deploy /opt/releases/current
```

Только группа (двоеточие в начале).

```bash
sudo chown :www-data /var/www/site/uploads
```

Владелец с пустым полем группы (группа — login-группа пользователя).

```bash
sudo chown nginx: /var/cache/nginx
```

Рекурсивно всё дерево.

```bash
sudo chown -R app:app /var/lib/myapp
```

Взять владельца и группу с другого файла (GNU).

```bash
sudo chown --reference=/etc/passwd /tmp/staged-file
```

Показать каждый изменённый объект.

```bash
sudo chown -v root:root /etc/myapp.conf
```

Несколько путей за один вызов.

```bash
sudo chown app:app /var/lib/myapp/data /var/lib/myapp/cache
```

## Синтаксис

```text
chown [опции] владелец[:группа] путь ...
chown [опции] :группа путь ...
```

Формат **`пользователь:группа`** задаёт оба поля; **`пользователь:`** (двоеточие в конце) или **`пользователь`** — только пользователя; **`:группа`** — только группу.

## Поведение и параметры

Смена владельца на произвольного пользователя обычно требует **прав суперпользователя** (`sudo`). Система проверяет, что указанные **имя пользователя** и **группа** существуют в локальных или сетевых учётных записях.

| Параметр | Что делает |
| --- | --- |
| `-R`, `--recursive` | Обойти каталог и всё внутри |
| `-h` | При символических ссылках менять саму ссылку, а не цель (поведение по умолчанию на многих системах — см. `man chown`) |
| `--reference=файл` | Взять владельца и группу с другого файла (GNU) |
| `-v` | Печатать, какие объекты изменились |

**Неправильный владелец** после распаковки архива или `cp` от root — частая причина недоступности файлов сервису. После `chown` имеет смысл проверить права через `ls -l` или `stat`.

## Примеры

Выдать дерево каталога приложению `app`, как в документации дистрибутива.

```bash
sudo chown -R app:app /var/lib/myapp
```

Понять текущего владельца перед правкой.

```bash
ls -l /etc/myapp.conf
sudo chown root:myapp /etc/myapp.conf
```
