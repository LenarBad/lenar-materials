# mv

## Назначение

**Переместить** файл или каталог в другое место или **переименовать** в том же каталоге. На одной файловой системе это часто быстрая операция (меняются имена в каталоге), между разделами фактически идёт копирование с удалением источника.

## TLDR

Переименовать файл.

```bash
mv draft.txt final.txt
```

Переместить файл в каталог.

```bash
mv report.pdf /tmp/
```

Несколько файлов в один каталог (последний аргумент — каталог).

```bash
mv a.log b.log /var/log/archive/
```

Спросить перед перезаписью.

```bash
mv -i todo.txt done.txt
```

Не перезаписывать существующий файл.

```bash
mv -n build.tmp build.out
```

Переместить только если источник новее (или цели нет).

```bash
mv -u sync/data.bin /mnt/backup/
```

Подробный вывод.

```bash
mv -v /opt/oldname /opt/newname
```

## Синтаксис

```text
mv [опции] источник ... назначение
```

Несколько **источников** допустимы только если **назначение** — существующий каталог.

## Поведение и параметры

Если **назначение** — существующий файл, он обычно **перезаписывается** (если не указаны **`-i`** или **`-n`**). Для каталогов поведение при совпадении имён зависит от того, существует ли каталог-назначение; при сомнении сначала **`ls -ld`** на оба пути.

| Параметр | Что делает |
| --- | --- |
| `-i`, `--interactive` | Подтверждать перезапись |
| `-n`, `--no-clobber` | Не перезаписывать существующий файл |
| `-u`, `--update` | Не трогать цель, если она новее источника |
| `-v`, `--verbose` | Печатать операции |

Тяжёлые деревья между дисками лучше переносить **`rsync`** с проверкой, затем удалять источник осознанно.

## Примеры

Переименовать каталог проекта.

```bash
mv ~/projects/app-v1 ~/projects/app
```
