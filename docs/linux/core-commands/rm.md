# rm

## Назначение

**Удалить** файлы и каталоги. Команда **необратима** в смысле «корзины»: восстановление возможно только из бэкапа или инструментов файловой системы. Частая причина инцидентов — **`rm -rf`** с опечаткой в пути; перед массовым удалением используйте **просмотр списка** (`ls`, `find … -print`).

## TLDR

Удалить один файл.

```bash
rm /tmp/olddump.sql
```

Интерактивно подтвердить каждый файл.

```bash
rm -i notes.txt draft.txt
```

Удалить пустой каталог (безопаснее, чем `rm -r`, если каталог точно пуст).

```bash
rmdir /tmp/emptydir
```

Рекурсивно удалить каталог с содержимым (осторожно: проверьте путь дважды).

```bash
rm -r /tmp/build-output
```

Принудительно, без вопросов (в скриптах; для интерактива опасно).

```bash
rm -f /tmp/lockfile.pid
```

Рекурсивно и без лишних вопросов — **только** если путь проверен и это не корень системы.

```bash
rm -rf /tmp/app-cache-sandbox
```

## Синтаксис

```text
rm [опции] путь ...
```

**`rmdir каталог`** удаляет только **пустой** каталог; для дерева нужен **`rm -r`** или пошаговая очистка.

## Поведение и параметры

**`-f`** подавляет часть диагностик и не спрашивает о несуществующих файлах — удобно в скриптах, но повышает риск молчаливого удаления не того. **`rm -rf /`** разрушителен; защитные приёмы в оболочке (алиасы `rm -i`) не заменяют внимательность.

| Параметр | Что делает |
| --- | --- |
| `-r`, `-R`, `--recursive` | Удалять каталоги и всё внутри |
| `-f`, `--force` | Не спрашивать, игнорировать отсутствующие имена (частично) |
| `-i` | Спрашивать перед каждым файлом |
| `-d`, `--dir` | Удалить пустой каталог (GNU; см. man) |
| `--one-file-system` | Не выходить на другие смонтированные ФС (GNU) |

Альтернатива **`find … -delete`**: сначала всегда запускайте то же условие с **`-print`** или **`-ls`**.

## Примеры

Сначала посмотреть, что удалится, затем удалить (два шага, без одной длинной строки с `-delete`).

```bash
find /var/tmp/myapp -type f -name "*.bak" -print
# проверили список
find /var/tmp/myapp -type f -name "*.bak" -delete
```
