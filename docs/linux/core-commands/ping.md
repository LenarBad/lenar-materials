# ping

## Назначение

Проверить **достижимость узла** по сети с помощью ICMP echo-request/reply (если протокол не заблокирован). Даёт задержку и потери пакетов; не заменяет проверку TCP/UDP-порта и приложений.

## TLDR

Отправить несколько пакетов и выйти.

```bash
ping -c 4 example.com
```

Пинг по IPv4 (если утилита различает семейства).

```bash
ping -4 10.0.1.1
```

Ограничить время ожидания ответа (секунды; детали в man на вашей ОС).

```bash
ping -c 2 -W 2 10.0.1.1
```

Бесконечный пинг до Ctrl+C (осторожно в скриптах).

```bash
ping example.com
```

Интервал между пакетами (не все версии поддерживают одинаково).

```bash
ping -c 10 -i 0.5 192.0.2.1
```

## Синтаксис

```text
ping [опции] узел
```

**Узел** — имя хоста или IP. Для имён сначала выполняется DNS-резолвинг.

## Поведение и параметры

Успешный `ping` означает лишь, что **ICMP доходит**; firewall может резать ICMP при открытом HTTP. Обратная ситуация тоже возможна: приложение недоступно, а ICMP отвечает.

| Параметр | Что делает |
| --- | --- |
| `-c N` | После N пакетов завершиться |
| `-W` / `-w` | Таймауты ожидания (имя и единицы зависят от реализации) |
| `-4` / `-6` | Принудительно IPv4 или IPv6 (где поддерживается) |
| `-I интерфейс` | Исходящий интерфейс (часто для многоадресных хостов) |

На некоторых системах `ping` требует **capabilities** или root для произвольного интервала; при «Operation not permitted» смотрите `man ping`.

## Примеры

Быстрая проверка шлюза.

```bash
ping -c 3 "$(ip route | awk '/default/ {print $3; exit}')"
```
