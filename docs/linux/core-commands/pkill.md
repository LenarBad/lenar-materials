# pkill

## Назначение

**Отправить сигнал процессам**, отобранным по **имени или шаблону** (как `pgrep`), без ручного поиска PID. Удобно, когда нужно завершить все подходящие процессы сразу; для одного известного PID проще **`kill`**. Для сервисов под **systemd** чаще предпочтительны **`systemctl stop`** / **`restart`**.

## TLDR

Завершить процессы по имени исполняемого файла (по умолчанию сигнал `TERM`).

```bash
pkill nginx
```

По полной командной строке (осторожно: широкий шаблон заденет лишнее).

```bash
pkill -f "python.*worker"
```

Явно указать сигнал.

```bash
pkill -TERM myworker
```

```bash
pkill -KILL runaway
```

Только процессы указанного пользователя.

```bash
pkill -u www-data -f php-fpm
```

Завершить только **самый новый** совпавший процесс.

```bash
pkill -n -f "myapp --port 8080"
```

## Синтаксис

```text
pkill [опции] шаблон
```

Семантика **шаблона** совпадает с **`pgrep`**: без **`-f`** сравнение идёт с именем процесса, с **`-f`** — со всей командной строкой.

## Поведение и параметры

Команда **находит** процессы так же, как `pgrep`, затем посылает им выбранный **сигнал**. Код завершения и сообщения зависят от того, было ли что-то завершено (см. `man pkill`).

| Параметр | Что делает |
| --- | --- |
| `-f`, `--full` | Сопоставлять шаблон с полной командной строкой |
| `-u`, `--euid` | Только процессы указанного пользователя (имя или uid) |
| `-n`, `--newest` | Выбрать только самый новый из совпавших |
| `-o`, `--oldest` | Выбрать только самый старый из совпавших |
| `-x`, `--exact` | Точное совпадение с именем процесса |
| `-signal` | Сигнал: `-TERM`, `-KILL`, `-HUP` или `-число` (как у `kill`) |

Перед массовым завершением имеет смысл сначала выполнить **`pgrep -af шаблон`** и убедиться, что в списке только нужные процессы.

## Примеры

Сначала посмотреть кандидатов, затем завершить.

```bash
pgrep -af mybatch
pkill -TERM -f "mybatch --job nightly"
```
