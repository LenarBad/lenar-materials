# mkdir

## Назначение

**Создать каталог** (один или несколько), при необходимости — всю цепочку родительских каталогов за раз. Нужна при развёртывании приложений, подготовке путей для логов и временных данных.

## TLDR

Один каталог в текущем месте.

```bash
mkdir backups
```

Несколько каталогов сразу.

```bash
mkdir tmp logs data
```

Создать вложенный путь целиком (если промежуточных каталогов ещё нет).

```bash
mkdir -p /var/lib/myapp/cache/sessions
```

Задать права при создании (восьмеричный режим).

```bash
mkdir -m 750 /etc/myapp/secrets.d
```

Подробный вывод.

```bash
mkdir -pv /srv/www/site/htdocs
```

## Синтаксис

```text
mkdir [опции] каталог ...
```

## Поведение и параметры

Если каталог **уже существует**, без **`-p`** команда обычно завершается с ошибкой; с **`-p`** такая ситуация не считается ошибкой (удобно в скриптах).

| Параметр | Что делает |
| --- | --- |
| `-p`, `--parents` | Создать недостающие родительские каталоги; не ругаться, если конечный каталог уже есть |
| `-m`, `--mode=режим` | Права сразу при создании (как в `chmod`, восьмеричный или символьный вид — см. `man mkdir`) |
| `-v`, `--verbose` | Печатать каждый созданный каталог |

## Примеры

Подготовить дерево под деплой и сразу выставить права.

```bash
mkdir -pm 755 /opt/myapp/{bin,lib,etc}
```
