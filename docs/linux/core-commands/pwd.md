# pwd

## Назначение

Показать **полный путь** от корня файловой системы до каталога, в котором вы сейчас находитесь (того, куда относится команда `cd` без аргументов).

## TLDR

Показать текущий каталог.

```bash
pwd
```

Путь с «физическими» именами каталогов (без обхода через символические ссылки).

```bash
pwd -P
```

Путь в логическом виде оболочки (удобно сравнивать с `-P` после `cd` через ссылку).

```bash
pwd -L
```

Подставить путь в сообщение или лог.

```bash
echo "Рабочая директория: $(pwd)"
```

Сохранить путь в переменную для скрипта.

```bash
here="$(pwd)"
```

Понять, встроенная ли `pwd` в оболочке или внешняя программа (bash).

```bash
type pwd
```

## Синтаксис

```text
pwd [опции]
```

## Поведение и параметры

**Без ключей** команда выводит один путь в виде строки.

**`-P` и `-L`** имеют смысл, если вы переходили в каталог **через символическую ссылку** (когда один каталог указывает на другой):

- **`-P`** — показать путь к **реальному** каталогу на диске (все звенья пути — настоящие имена).
- **`-L`** — показать путь **в том виде, в каком его ведёт оболочка** (иногда в пути остаётся имя ссылки). Для встроенной в оболочку `pwd` это часто режим по умолчанию.

| Параметр | Что делает |
| --- | --- |
| `-L` | Логический путь (как у оболочки) |
| `-P` | Физический путь (без «обходных» имён через ссылки) |
| `--help`, `--version` | Справка и версия (у отдельной программы `pwd` из пакета coreutils в Linux) |

В терминале обычно запускается **встроенная** `pwd` из вашей оболочки. Отдельная программа `/usr/bin/pwd` может дать тот же результат, но детали по умолчанию стоит сверять с `man pwd` на вашей системе.

## Примеры

Сохранить путь в переменную; кавычки нужны, если в пути могут быть пробелы.

```bash
dir="$(pwd)"
```

После `cd` через символическую ссылку сравните два варианта.

```bash
pwd -L
pwd -P
```
