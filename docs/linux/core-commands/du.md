# du

## Назначение

Оценить **сколько места** занимают файлы и каталоги на диске. Используют при поиске «тяжёлых» логов, кэшей и каталогов при ошибке **No space left on device** и при планировании очистки.

## TLDR

Итоговый размер одного каталога «человекочитаемо».

```bash
du -sh /var/log
```

Размер текущего каталога целиком.

```bash
du -sh .
```

По каждому элементу в текущем каталоге (без ухода глубже).

```bash
du -sh *
```

Один уровень вложенности и сортировка по размеру (ищем «тяжёлых детей»).

```bash
du -h --max-depth=1 /var/lib | sort -h
```

Сумма по нескольким путям и общая строка `total`.

```bash
du -shc /var/log /var/tmp
```

Не выходить за пределы одной файловой системы (без учёта смонтированных внутри).

```bash
du -shx /var
```

Все файлы и каталоги с размерами под деревом (много строк).

```bash
du -ah /var/log | sort -h | tail
```

Только «верхний» уровень `/home` без подкаталогов пользователей вглубь.

```bash
du -h --max-depth=1 /home | sort -h
```

Показать время изменения вместе с размером (GNU; формат см. `man du`).

```bash
du -h --time /var/log
```

## Синтаксис

```text
du [опции] [путь ...]
```

Без путей часто обрабатывается текущий каталог.

## Поведение и параметры

Команда **суммирует** размер файлов внутри указанных путей. Для каталога без ключей вывод может быть **длинным** (по каждому подкаталогу); ключ **`-s`** (summary) даёт **одну строку** на каждый переданный путь.

| Параметр | Что делает |
| --- | --- |
| `-h`, `--human-readable` | Размеры в КиБ, МиБ, ГиБ |
| `-s`, `--summarize` | Только итог по каждому аргументу, без детализации вглубь |
| `--max-depth=N` | Показать вложенность не глубже N (GNU) |
| `-a`, `--all` | Учитывать и файлы, не только каталоги (много строк) |
| `-x`, `--one-file-system` | Не заходить на другие смонтированные файловые системы |
| `-c`, `--total` | В конце строка `total` |
| `--time` | Показать время последнего изменения (GNU; уточните формат в `man du`) |

**Отличие от `df`:** `df` смотрит на **заполненность разделов** целиком; `du` — на **сумму файлов** по путям. Расхождения возможны из‑за открытых удалённых файлов, других точек монтирования внутри пути и особенностей подсчёта.

## Примеры

Найти самые крупные подкаталоги в `/var/log` (верхние строки после сортировки).

```bash
du -h --max-depth=1 /var/log | sort -h | tail
```

Проверить размер домашнего каталога пользователя.

```bash
du -sh /home/lenar
```
