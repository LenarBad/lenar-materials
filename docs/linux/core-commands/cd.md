# cd

## Назначение

**Сменить текущий рабочий каталог** оболочки: куда относится относительный путь в командах и куда ведёт `pwd` без аргументов. Обычно это **встроенная команда** (`builtin`) в bash/zsh, а не отдельный файл `/bin/cd` — поведение смотрите в `help cd` (bash) или в документации своей оболочки.

## TLDR

Перейти в каталог (абсолютный или относительный путь).

```bash
cd /var/log
```

В домашний каталог текущего пользователя.

```bash
cd
```

На уровень выше.

```bash
cd ..
```

Вернуться в предыдущий каталог (обмен с текущим).

```bash
cd -
```

Перейти в каталог другого пользователя (если есть доступ).

```bash
cd ~postgres
```

## Синтаксис

```text
cd [каталог]
```

Аргумент **`-`** — переключение с предыдущего каталога; путь к **дому** задают `~`, `~имя` или пустой вызов `cd`.

## Поведение и параметры

После успешного `cd` обновляются переменные вроде **`PWD`** и **`OLDPWD`** (в типичных оболочках). Для скриптов иногда пишут **`cd … || exit`**, чтобы не продолжать работу в неверном каталоге при ошибке.

| Форма | Что делает |
| --- | --- |
| `cd путь` | Сделать `путь` текущим каталогом |
| `cd` / `cd ~` | Домашний каталог пользователя |
| `cd -` | Предыдущий каталог (удобно переключаться туда-обратно) |
| `cd ..` | Родительский каталог |
| `CDPATH` | Если задана, оболочка может искать относительный `каталог` по списку префиксов — см. документацию оболочки |

Ошибка «Permission denied» или «Not a directory» означает проблему с правами или типом объекта; **`pwd -P`** помогает убедиться, где вы оказались после переходов по ссылкам.

## Примеры

Скрипт: зайти в каталог проекта или выйти с ошибкой.

```bash
cd /srv/myapp/current || exit 1
```
