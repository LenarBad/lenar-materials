# ps

## Назначение

Показать **снимок процессов**: кто запущен, под каким пользователем, сколько потребляет CPU/RAM, какая полная командная строка. Базовая команда для разборов нагрузки и отладки «что за PID».

## TLDR

Процессы текущего сеанса в простом виде.

```bash
ps
```

Все процессы с подробностями (стиль BSD).

```bash
ps aux
```

Детально по одному PID.

```bash
ps -fp 1234
```

Выбрать поля и отсортировать по загрузке CPU.

```bash
ps -eo pid,cmd,%cpu,%mem --sort=-%cpu | head
```

Дерево процессов.

```bash
ps auxf
```

## Синтаксис

```text
ps [опции]
```

Синтаксис **BSD**, **System V** и **GNU** отличаются: на Linux обычно работают и `ps aux`, и `ps -ef`; для переносимых скриптов сверяйтесь с `man ps`.

## Поведение и параметры

Без ключей `ps` часто показывает процессы **текущего терминала**. Для обзора всей системы используют **`aux`** или **`eo`** с нужными полями.

| Параметр / форма | Что делает |
| --- | --- |
| `aux` | Расширенный список процессов (BSD-стиль) |
| `-ef` | Полный список в стиле System V |
| `-fp PID` | Фильтр по PID с «полным» форматом |
| `-eo поля` | Задать набор колонок (`pid`, `cmd`, `%cpu`, `%mem`, `etime` и др.) |
| `--sort=-%cpu` | Сортировка по полю (GNU `ps`) |

Колонка **`CMD`** / **`COMMAND`** может быть усечена; полную строку иногда удобнее смотреть через `pgrep -af` или `/proc/PID/cmdline`.

## Примеры

Найти тяжёлые по памяти процессы среди первых строк.

```bash
ps -eo pid,cmd,%mem --sort=-%mem | head -n 15
```
