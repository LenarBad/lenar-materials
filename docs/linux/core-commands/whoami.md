# whoami

## Назначение

Показать **имя пользователя**, под которым сейчас выполняется ваша оболочка (и команды в ней). После `sudo`, `su` или входа по SSH это помогает убедиться, что вы действительно тот пользователь, за кого себя принимаете.

## TLDR

Имя пользователя, от имени которого выполняется оболочка (одна строка).

```bash
whoami
```

Сохранить имя в переменную.

```bash
me="$(whoami)"
```

Проверить, что вы не root, перед опасной операцией.

```bash
test "$(whoami)" != "root" && echo "Ок, не root"
```

То же имя часто даёт `id`; сравните при сомнениях.

```bash
whoami
id -un
```

Версия утилиты (GNU coreutils).

```bash
whoami --version
```

## Синтаксис

```text
whoami [опции]
```

Аргументов с путями к файлам нет — только ключи, если они есть в вашей версии программы.

## Поведение и параметры

Команда выводит имя пользователя, под которым в данный момент идёт работа процесса (обычно это совпадает с тем, что в кратком виде показывает `id`).

Чем отличается от похожих команд:

| Команда | Зачем смотреть |
| --- | --- |
| `whoami` | Только имя пользователя — удобно в скриптах |
| `id` | Числовые идентификаторы, группы — когда нужны детали |
| `logname` | Имя входа в систему — в необычных сессиях может отличаться |

| Параметр | Что делает (типично в Linux с GNU coreutils) |
| --- | --- |
| `--help` | Краткая справка |
| `--version` | Версия пакета |

Если имя «не то», что ожидали, смотрите также `id` и настройки входа (`su`, `sudo`, контейнер). На сложных установках с централизованными учётными записями при сбоях смотрят учётные записи в системе и журналы службы входа — это уже не задача одной строки `whoami`.

## Примеры

Проверка в скрипте перед действием от имени root.

```bash
if [ "$(whoami)" = "root" ]; then
  echo "Запуск от root"
fi
```
