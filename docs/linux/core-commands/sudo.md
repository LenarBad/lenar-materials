# sudo

## Назначение

**Выполнить одну команду** (или зайти в интерактивную оболочку) **от имени другого пользователя** — чаще всего суперпользователя — по правилам из `/etc/sudoers` и файлов в `/etc/sudoers.d/`. Это основной способ администрирования без постоянного входа под root: права выдают точечно, действия можно аудировать.

## TLDR

Одна команда с правами суперпользователя (запросит пароль, если настроено).

```bash
sudo systemctl restart nginx
```

Открыть интерактивную оболочку как целевой пользователь (часто root): окружение ближе к «полноценному входу», чем у одиночной команды.

```bash
sudo -i
```

Выполнить команду от имени другого пользователя.

```bash
sudo -u www-data id
```

Показать, какие команды вам разрешено запускать через sudo (и от чьего имени).

```bash
sudo -l
```

Подробный разбор правил в длинном формате.

```bash
sudo -ll
```

Редактировать безопасно конфигурацию sudo (блокировка и проверка синтаксиса).

```bash
sudo visudo
```

Не спрашивать пароль в скрипте — **только** если в политике это явно разрешено для этой команды (иначе команда завершится ошибкой).

```bash
sudo -n true
```

Сохранить текущее окружение переменных, если политика позволяет (осторожно с путями к бинарникам).

```bash
sudo -E editor /etc/myapp/config.yml
```

Домашний каталог целевого пользователя в переменной `HOME` (удобно для программ, пишущих в `~`).

```bash
sudo -H -u backup rsync -a /data/ ~backup/mirror/
```

## Синтаксис

```text
sudo [опции] [команда [аргументы ...]]
```

Без **команды** поведение зависит от опций (например **`-i`** / **`-s`**) и настроек; уточняйте в `man sudo` на своей системе.

## Поведение и параметры

Кто может что выполнять, задаётся в **sudoers** (не редактируйте напрямую обычным редактором без блокировки — используйте **`visudo`**). Ошибка в синтаксисе может **лишить вас sudo** до исправления с консоли recovery.

| Параметр | Что делает |
| --- | --- |
| `-l`, `--list` | Показать разрешённые для вас команды (или сообщить, что запрещено) |
| `-i`, `--login` | Запустить login-оболочку целевого пользователя (часто root) |
| `-s`, `--shell` | Запустить оболочку из переменной `SHELL` (или по умолчанию) |
| `-u пользователь` | Выполнить от имени указанного пользователя |
| `-E`, `--preserve-env` | Сохранить окружение (если политика и список переменных позволяют) |
| `-H`, `--set-home` | Установить `HOME` каталог целевого пользователя |
| `-n`, `--non-interactive` | Не спрашивать пароль; при необходимости пароля — ошибка |
| `-v`, `--validate` | Обновить кэш времени (без запуска команды) |

**Безопасность:** выдавайте минимально необходимые команды, избегайте бездумного `(ALL) NOPASSWD: ALL`. После правок sudoers проверяйте вход и **`sudo -l`**. В скриптах предпочтительны **явные пути** к бинарникам (`/usr/bin/systemctl`), а не полагание на `PATH` злоумышленника.

## Примеры

Проверить, что вы действительно выполняете команду от root, перед опасным действием.

```bash
sudo id
sudo whoami
```
