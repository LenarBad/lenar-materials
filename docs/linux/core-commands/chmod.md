# chmod

## Назначение

Изменить **права доступа** к файлу или каталогу: кто может читать, записывать и выполнять. Нужна при настройке конфигов, веб-корней, скриптов и при исправлении «слишком открытых» или «закрытых» прав после копирования.

## TLDR

Восьмеричный режим: конфиг только владельцу и группе, остальные без доступа.

```bash
chmod 640 /etc/myapp.conf
```

Обычный файл: владелец читает/пишет, остальные только читают.

```bash
chmod 644 /var/www/html/index.html
```

Только владелец читает и пишет (секреты, ключи).

```bash
chmod 600 ~/.ssh/id_rsa
```

Каталог с исполняемыми файлами, рекурсивно.

```bash
chmod -R 755 /opt/myapp/bin
```

Символьно: добавить выполнение владельцу.

```bash
chmod u+x /opt/myapp/run.sh
```

Символьно: убрать запись у группы и остальных.

```bash
chmod go-w /var/www/html/index.html
```

Дать всем право на чтение.

```bash
chmod a+r /usr/local/share/doc/readme.txt
```

Рекурсивно скопировать права с эталонного файла (GNU).

```bash
chmod --reference=/etc/passwd /tmp/copy.conf
```

Подробный вывод: что именно изменилось.

```bash
chmod -v u+x script.sh
```

## Синтаксис

```text
chmod [опции] режим путь ...
chmod [опции] восьмеричный_режим путь ...
```

**Режим** — либо **восьмеричное число** (например `644`), либо **символьная запись** (`u+rwx`, `go-w`).

## Поведение и параметры

Права описывают три категории: **владелец (u)**, **группа (g)**, **остальные (o)**; буква **a** означает всех сразу. Для каждой категории задают **чтение (r)**, **запись (w)**, **выполнение (x)**; для каталогов «выполнение» означает право **зайти** в каталог.

**Восьмеричный вид** — три цифры (иногда четыре с учётом спецбитов, это уже отдельная тема): первая — владелец, вторая — группа, третья — остальные. Цифра — сумма: 4 = чтение, 2 = запись, 1 = выполнение (например `6` = `rw-`).

| Параметр | Что делает |
| --- | --- |
| `-R`, `--recursive` | Применить ко всем файлам и подкаталогам по пути |
| `--reference=файл` | Взять те же права, что у указанного файла (GNU coreutils) |
| `-v` | Показать, что изменилось (verbose) |

**Символьная запись:** `категория(и) операция права`, например `g+w` (группе добавить запись), `o-r` (остальным убрать чтение), `a+X` (добавить «x» только там, где это уместно для каталогов и исполняемых файлов — уточняйте в `man chmod` на своей системе).

Менять права на чужие файлы обычно может только **владелец** или **суперпользователь**. Ошибка «Permission denied» часто решается через `sudo`, но сначала убедитесь, что меняете **нужный** путь.

## Примеры

Оставить конфиг только владельцу и группе, без доступа остальным.

```bash
chmod 660 /etc/myapp/secrets.env
```

Убрать запись у группы и у всех остальных, не трогая владельца (символьно).

```bash
chmod go-w /var/www/html/index.html
```
