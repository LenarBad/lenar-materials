# grep

## Назначение

**Найти строки**, совпадающие с шаблоном, в файлах или во входном потоке. Используют при просмотре логов, поиске настроек в `/etc`, фильтрации вывода других команд. Расширенные регулярные выражения задают флагом **`-E`** (часто удобнее, чем «классический» синтаксис).

## TLDR

Найти строки с текстом в одном файле.

```bash
grep Failed /var/log/myapp/app.log
```

Регистронезависимый поиск.

```bash
grep -i error /var/log/syslog
```

Номера строк.

```bash
grep -n listen /etc/nginx/nginx.conf
```

Рекурсивно по каталогу (осторожно с большими деревьями).

```bash
grep -r "PermitRootLogin" /etc/ssh
```

Расширенные регулярные выражения.

```bash
grep -E "^(ERR|WARN)" /var/log/myapp/*.log
```

Только имена файлов с совпадениями.

```bash
grep -l ERROR /var/log/myapp/*.log
```

Фильтр вывода другой команды.

```bash
journalctl -u nginx --no-pager | grep -i error
```

## Синтаксис

```text
grep [опции] шаблон [файл ...]
```

Без **файла** `grep` читает **стандартный ввод**.

## Поведение и параметры

По умолчанию шаблон — **базовые** регулярные выражения; **`-E`** включает **расширенный** синтаксис, **`-F`** — поиск **подстроки** без спецсимволов. Поведение и ключи на BSD/macOS и GNU могут отличаться; для переносимых скриптов сверяйтесь с `man grep` на целевой системе.

| Параметр | Что делает |
| --- | --- |
| `-i`, `--ignore-case` | Без учёта регистра |
| `-n`, `--line-number` | Номер строки |
| `-r`, `-R`, `--recursive` | Обход каталогов |
| `-E` | Расширенные регулярные выражения |
| `-F`, `--fixed-strings` | Буквальная строка, не шаблон |
| `-l`, `--files-with-matches` | Только имена файлов с совпадениями |
| `-v`, `--invert-match` | Строки **без** совпадения |
| `--color=auto` | Подсветка (если поддерживается) |

Для очень больших логов быстрее бывают специализированные инструменты или **`rg`** (если установлен), но **`grep`** остаётся базовым стандартом.

## Примеры

Найти открытые порты в выводе `ss` и отфильтровать по номеру.

```bash
ss -tlnp | grep ':443'
```
