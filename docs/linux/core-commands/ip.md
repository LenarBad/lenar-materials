# ip

## Назначение

Показать и настраивать **сетевые интерфейсы, адреса и маршруты** (пакет **iproute2**). Современная замена части сценариев `ifconfig` и `route` на Linux.

## TLDR

Адреса и интерфейсы.

```bash
ip a
```

Кратко: только IP по интерфейсам.

```bash
ip -br a
```

Таблица маршрутов.

```bash
ip route
```

Маршрут до конкретного узла.

```bash
ip route get 93.184.216.34
```

Соседи (ARP/NDP).

```bash
ip neigh
```

Список сетевых пространств имён (если используются).

```bash
ip netns list
```

## Синтаксис

```text
ip [опции] объект [команда [аргументы]]
```

Объекты: **`link`**, **`address`**, **`route`**, **`neigh`**, **`rule`** и др. Краткие формы вроде **`ip a`** = **`ip address`**.

## Поведение и параметры

Просмотр обычно доступен без root; **изменение** адресов и маршрутов — от привилегированного пользователя.

| Подкоманда / опция | Что делает |
| --- | --- |
| `a` / `address` | IPv4/IPv6-адреса на интерфейсах |
| `link` | Состояние интерфейсов (UP/DOWN, MTU) |
| `route` | Таблица маршрутизации |
| `neigh` | Кэш соседей L2/L3 |
| `-br`, `-brief` | Компактный вывод (удобно в скриптах) |
| `-4` / `-6` | Только IPv4 или IPv6 |

Для постоянной конфигурации в продакшене часто используют **NetworkManager**, **netplan**, **ifupdown** и т.д., а `ip` — для оперативной диагностики и временных правок.

## Примеры

Найти default gateway в первой строке default-маршрута.

```bash
ip route show default
```
