# curl

## Назначение

Передавать данные по **URL**: HTTP(S), редиректы, заголовки, тело ответа. Незаменима для проверки API, TLS-сертификатов, кодов ответа и таймаутов без браузера.

## TLDR

Показать тело ответа со страницы.

```bash
curl "$SERVICE_URL"
```

Только заголовки ответа (HEAD-запрос по сути).

```bash
curl -I "$SERVICE_URL"
```

Подробный вывод: TLS, запрос, заголовки (отладка).

```bash
curl -vk "$SERVICE_URL/health"
```

Следовать редиректам.

```bash
curl -L "$SHORT_URL"
```

Сохранить вывод в файл.

```bash
curl -fsS -o /tmp/page.html "$SERVICE_URL"
```

POST с телом JSON и заголовком Content-Type.

```bash
curl -X POST -H 'Content-Type: application/json' -d '{"ok":true}' "$API_URL/v1/ping"
```

## Синтаксис

```text
curl [опции] URL ...
```

Можно указать **несколько URL**; опции применяются к запросам согласно документации.

## Поведение и параметры

По умолчанию `curl` выполняет **GET**. Коды **`2xx`** обычно успех; **`4xx`** — ошибка клиента; **`5xx`** — сервера. Для скриптов полезны **`--fail`** / **`--fail-with-body`** и явные таймауты.

| Параметр | Что делает |
| --- | --- |
| `-v`, `--verbose` | Подробный протокол (включая TLS handshake при HTTPS) |
| `-I`, `--head` | Запросить только заголовки |
| `-k`, `--insecure` | Не проверять сертификат TLS (только диагностика) |
| `-L` | Следовать редиректам |
| `-d данные` | Тело запроса (часто с `-X POST`) |
| `-H 'заголовок'` | Добавить HTTP-заголовок |
| `-o файл` | Записать тело в файл |
| `-fsS` | Тихий режим + показ ошибок (частая связка в cron) |

**`-k`** снижает безопасность: используйте только для локальной проверки self-signed.

## Примеры

Проверить health-endpoint и HTTP-код.

```bash
curl -sS -o /dev/null -w "%{http_code}\n" "$API_URL/health"
```
