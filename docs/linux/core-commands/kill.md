# kill

## Назначение

**Отправить сигнал** процессу по PID: обычно — попросить завершиться (`TERM`) или принудительно остановить (`KILL`). Используют, когда сервис не отвечает, скрипт завис, или нужно перечитать конфигурацию (для некоторых демонов — по документации, часто через `systemctl`, а не «голый» `kill`).

## TLDR

Мягко попросить процесс завершиться (сигнал по умолчанию часто `TERM`).

```bash
kill 1234
```

Явно отправить `SIGTERM` (корректное завершение).

```bash
kill -TERM 1234
```

Краткая форма номера сигнала для `TERM`.

```bash
kill -15 1234
```

Принудительное завершение, если на `TERM` не отреагировал.

```bash
kill -KILL 1234
```

Отправить `HUP` (часто — перечитать конфиг; зависит от программы).

```bash
kill -HUP 1234
```

Показать список имён и номеров сигналов.

```bash
kill -l
```

## Синтаксис

```text
kill [опции] PID ...
```

Можно указать **несколько PID**. Для процессов под **systemd** сначала проверьте `systemctl status` и логи — остановка через unit часто безопаснее.

## Поведение и параметры

Сигнал **`TERM` (15)** даёт процессу шанс завершить работу и снять ресурсы. **`KILL` (9)** обрабатывается ядром: процесс не может перехватить сигнал — используйте только если мягкий вариант не сработал.

| Сигнал | Номер | Типичное применение |
| --- | --- | --- |
| `TERM` | 15 | Нормальное завершение (первый выбор) |
| `KILL` | 9 | Жёсткая остановка |
| `HUP` | 1 | Перезагрузка конфига или закрытие сессии (зависит от приложения) |
| `INT` | 2 | Как Ctrl+C в терминале |

`kill` требует **прав**: обычно можно слать сигналы своим процессам; чужим — чаще только от root. Ошибка «Operation not permitted» ожидаема без `sudo`.

## Примеры

Подождать несколько секунд после `TERM`, затем при необходимости `KILL`.

```bash
kill -TERM 1234
sleep 3
kill -KILL 1234
```
