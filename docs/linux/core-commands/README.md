## Компактный справочник

Полный синтаксис: `man <команда>` / `<команда> --help`.

## Basics

### whoami

показать имя текущего пользователя. **[Подробное описание →](whoami.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| _(без параметров)_ | вывод текущего пользователя | Вывод (пример): `lenar` |

```bash
whoami
```

### pwd

показать текущую директорию. **[Подробное описание →](pwd.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-P` | физический путь без symlink | Вывод (пример): `/var/www/app` |
| _(без параметров)_ | путь shell-контекста | Вывод (пример): `/home/lenar/project` |

```bash
pwd
```

```bash
pwd -P
```

### cd

сменить текущий каталог. **[Подробное описание →](cd.md)**

| Форма | Что делает | Пояснение |
| --- | --- | --- |
| `cd путь` | перейти в каталог | Относительный путь считается от текущего каталога |
| `cd` / `cd ~` | домашний каталог | Эффект: как правило `$HOME` |
| `cd -` | предыдущий каталог | Эффект: переключение с `OLDPWD` |

```bash
cd /var/log
```

```bash
cd -
```

### hostname

показать имя/адреса хоста. **[Подробное описание →](hostname.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-f` | показать FQDN | Вывод (пример): `srv-01.example.local` |
| `-I` | показать IP-адреса | Вывод (пример): `10.0.1.15 172.17.0.1` |

```bash
hostname
```

```bash
hostname -f
```

```bash
hostname -I
```

## Файлы и каталоги

### ls

список файлов и директорий. **[Подробное описание →](ls.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-l` | подробный формат | В выводе: строка с правами, владельцем, размером, именем файла |
| `-a` | показывать скрытые | В выводе появляются имена вроде `.env`, `.gitignore` |
| `-h` | человекочитаемые размеры | В выводе размеры как `4.0K`, `12M`, `2.1G` вместо сырых байт |

```bash
ls -lah
```

```bash
ls -lah /etc
```

### cp

копировать файлы и каталоги. **[Подробное описание →](cp.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-a` | архивный режим (рекурсивно + сохранить атрибуты, GNU) | Эффект: удобно для снимка дерева (см. `man cp`) |
| `-r` / `-R` | рекурсивно | Эффект: копировать подкаталоги |
| `-i` | спросить перед перезаписью | Эффект: безопаснее при совпадении имён в назначении |
| `-n` | не перезаписывать | Эффект: существующий файл в назначении остаётся |

```bash
cp -a /srv/app /var/backups/app-snapshot
```

```bash
cp -i ~/.bashrc ~/.bashrc.bak
```

### mv

переместить или переименовать. **[Подробное описание →](mv.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-i` | спросить перед перезаписью | Эффект: безопаснее при совпадении имён |
| `-n` | не перезаписывать цель | Эффект: существующий файл остаётся |
| `-u` | только если источник новее | Эффект: не затирать более новую цель |

```bash
mv draft.txt final.txt
```

```bash
mv -i report.pdf /tmp/
```

### mkdir

создать каталоги. **[Подробное описание →](mkdir.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-p` | создать родителей, не ошибаться если есть | Эффект: `mkdir -p a/b/c` создаёт всю цепочку |
| `-m` | режим прав при создании | Аргумент: например `750` или `u=rwx,g=rx,o=` |
| `-v` | подробный вывод | Эффект: печать каждого созданного пути |

```bash
mkdir -p /var/lib/myapp/cache
```

```bash
mkdir -m 750 /etc/myapp/secrets.d
```

### rm

удалить файлы и каталоги. **[Подробное описание →](rm.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-r` | рекурсивно | Эффект: удалить дерево; проверяйте путь перед запуском |
| `-f` | без вопросов | Эффект: опасно в связке с `-r` при опечатке |
| `rmdir` | только пустой каталог | Команда `rmdir`, не `rm`: безопаснее для одного пустого каталога |

```bash
rm -i notes.txt
```

```bash
rm -rf /tmp/build-output
```

### find

поиск по условиям. **[Подробное описание →](find.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-type` | тип объекта | Аргумент: `f` (файл) или `d` (каталог) |
| `-name` | шаблон имени | Аргумент: `"*.conf"` |
| `-mtime` | возраст в днях | Аргумент: `+7` (старше 7 дней), `-1` (моложе суток) |
| `-maxdepth` | глубина обхода | Аргумент: `2` |

```bash
find /etc -type f -name "*.conf"
```

```bash
find /var/log -type f -mtime +7
```

### rsync

синхронизация директорий. **[Подробное описание →](rsync.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-a` | архивный режим | Эффект: рекурсивно, с правами/временем и т.п. (см. `man rsync`) |
| `--delete` | удалить лишнее в destination | Эффект: в `dst/` удаляется то, чего нет в `src/` |
| `--dry-run` | preview без изменений | Эффект: только список действий, на диск ничего не пишется |

```bash
rsync -a --delete --dry-run src/ dst/
```

```bash
rsync -a --delete src/ dst/
```

### chmod

изменить права доступа. **[Подробное описание →](chmod.md)**

| Параметр/значение | Что делает | Пояснение |
| --- | --- | --- |
| `640` | `rw- r-- ---` | Аргумент в `chmod`: восьмеричный режим `640` (владелец rw, группа r, остальные нет) |
| `755` | `rwx r-x r-x` | Аргумент: `755` — типично для каталога или исполняемого файла |
| `-R` | рекурсивно | Эффект: то же правило прав ко всему дереву ниже пути |

```bash
chmod 640 /etc/myapp.conf
```

```bash
chmod -R 755 /opt/myapp/bin
```

### chown

изменить владельца/группу. **[Подробное описание →](chown.md)**

| Параметр/значение | Что делает | Пояснение |
| --- | --- | --- |
| `user:group` | новый владелец и группа | Аргумент: `root:root` или `app:app` после `chown` |
| `-R` | рекурсивно | Эффект: владелец/группа для всех вложенных файлов |

```bash
chown root:root /etc/myapp.conf
```

```bash
chown -R app:app /var/lib/myapp
```

### stat

метаданные файла. **[Подробное описание →](stat.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-c` | пользовательский формат вывода | Аргумент: строка формата, напр. `'%a %U:%G %n'` |
| _(без параметров)_ | полный отчет | Вывод: многострочный отчет (inode, права, время, размер) |

```bash
stat /etc/myapp.conf
```

```bash
stat -c "%a %U:%G %n" /etc/myapp.conf
```

## Текст и логи

### grep

поиск строк по шаблону. **[Подробное описание →](grep.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-i` | без учёта регистра | Эффект: `error` и `ERROR` совпадут |
| `-n` | номера строк | Эффект: удобно ссылаться на конфиг |
| `-r` | рекурсивно по каталогу | Эффект: осторожно на больших деревьях |
| `-E` | расширенные регулярные выражения | Эффект: богаче синтаксис шаблона |

```bash
grep -n listen /etc/nginx/nginx.conf
```

```bash
grep -i error /var/log/syslog
```

### tail

хвост файла и поток новых строк. **[Подробное описание →](tail.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-n` | последние N строк | Аргумент: `-n 100` |
| `-f` | ждать новые строки | Эффект: как `tail -f` для лога |
| `-F` | как `-f`, с переподключением | Эффект: удобнее при ротации логов |

```bash
tail -n 50 /var/log/nginx/error.log
```

```bash
tail -F /var/log/myapp/app.log
```

## Процессы и сервисы

### pgrep

найти процессы по имени/паттерну. **[Подробное описание →](pgrep.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-a` | показать командную строку | Вывод (пример): `1234 nginx: master process ...` |
| `-f` | искать по полной command line | Аргумент: шаблон в команде, напр. `python.*worker` |

```bash
pgrep -af nginx
```

```bash
pgrep -af "python.*worker"
```

### ps

детали процессов. **[Подробное описание →](ps.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-fp <PID>` | полный формат по PID | Команда: `ps -fp 1234` (подставьте свой PID) |
| `-eo ...` | выбрать поля вывода | Аргумент: список полей, напр. `pid,cmd,%cpu,%mem` |
| `--sort=-%cpu` | сортировка по CPU | Эффект: строки отсортированы по загрузке CPU (сначала самые тяжелые) |

```bash
ps -fp 1234
```

```bash
ps -eo pid,cmd,%cpu,%mem --sort=-%cpu | head
```

### kill

отправка сигналов процессу. **[Подробное описание →](kill.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-TERM` | мягкое завершение | Команда: `kill -TERM 1234` — процессу дают корректно завершиться |
| `-KILL` | принудительное завершение | Команда: `kill -KILL 1234` — без graceful, только если `-TERM` не помог |

```bash
kill -TERM 1234
```

```bash
sleep 3
```

```bash
kill -KILL 1234
```

### pkill

завершить процессы по имени/шаблону. **[Подробное описание →](pkill.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-f` | искать по полной command line | Эффект: как при `pgrep -f` — шире, чем по одному имени |
| `-u` | только пользователь | Аргумент: `www-data` или uid |
| `-TERM` / `-KILL` | какой сигнал послать | По умолчанию обычно `TERM`; `KILL` — последняя мера |

```bash
pkill -TERM nginx
```

```bash
pkill -f "python.*worker"
```

### systemctl

управление unit в systemd. **[Подробное описание →](systemctl.md)**

| Подкоманда | Что делает | Пояснение |
| --- | --- | --- |
| `status` | состояние unit | Команда: `systemctl status nginx` (подставьте имя unit) |
| `restart` | перезапуск | Команда: `systemctl restart nginx` |
| `enable` | автозапуск при boot | Эффект: unit стартует при загрузке (см. также `disable`) |
| `cat` | итоговый unit + override | Вывод: итоговый unit-файл и drop-in из `/etc/systemd/system` |

```bash
systemctl status nginx
```

```bash
sudo systemctl restart nginx
```

```bash
systemctl cat nginx
```

### journalctl

просмотр логов systemd. **[Подробное описание →](journalctl.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-u` | фильтр по unit | Аргумент: `-u nginx` или `-u myapp.service` |
| `-n` | последние N строк | Аргумент: `-n 100` |
| `--since` | от времени | Аргумент: `--since "2026-04-10 10:20:00"` (формат см. `man journalctl`) |
| `--no-pager` | без pager | Эффект: весь вывод сразу в stdout (удобно в pipe/скриптах) |

```bash
journalctl -u nginx -n 100 --no-pager
```

```bash
journalctl -u myapp --since "2026-04-10 10:20:00" --no-pager
```

## Сеть

### dig

DNS-запросы. **[Подробное описание →](dig.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `+short` | короткий ответ | Вывод (пример): одна строка с IP, напр. `93.184.216.34` |
| `@<dns-server>` | конкретный DNS-сервер | Аргумент в команде: `@1.1.1.1` перед именем зоны |

```bash
dig example.com +short
```

```bash
dig @1.1.1.1 example.com
```

### ping

проверка сетевой достижимости. **[Подробное описание →](ping.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-c` | число пакетов | Аргумент: `-c 4` — отправить 4 echo-request и выйти |
| `-W` | timeout ответа (сек) | Аргумент: `-W 2` — ждать ответ до 2 с (зависит от версии `ping`) |

```bash
ping -c 4 example.com
```

```bash
ping -c 2 -W 2 10.0.1.1
```

### nc

проверка TCP/UDP порта. **[Подробное описание →](nc.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-z` | scan mode без передачи данных | Эффект: проверка порта без полноценной сессии данных |
| `-v` | подробный вывод | В выводе: строки вроде `succeeded` или `Connection refused` |
| `-w` | timeout соединения | Аргумент: `-w 3` или `-w3` — секунды ожидания (синтаксис зависит от `nc`) |

```bash
nc -zv example.com 443
```

```bash
nc -zvw3 10.0.1.20 5432
```

### curl

проверка HTTP/TLS endpoint. **[Подробное описание →](curl.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-v` | подробный HTTP/TLS диалог | Вывод: запрос/ответ, заголовки, этапы TLS |
| `-k` | игнорировать TLS-сертификат | Эффект: не падать на self-signed (только для диагностики) |
| `-I` | только headers | Вывод (пример): первая строка `HTTP/1.1 200 OK` и заголовки |

```bash
curl -vk "$SERVICE_URL/health"
```

```bash
curl -I "$SERVICE_URL"
```

### ip

интерфейсы и маршруты. **[Подробное описание →](ip.md)**

| Подкоманда | Что делает | Пояснение |
| --- | --- | --- |
| `a` | адреса интерфейсов | Команда: `ip a` — список интерфейсов и адресов |
| `route` | таблица маршрутов | Команда: `ip route` — default и прочие маршруты |

```bash
ip a
```

```bash
ip route
```

### ss

сокеты и слушающие порты. **[Подробное описание →](ss.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-t` | TCP сокеты | Аргумент: флаг `-t` в наборе, напр. `ss -tulpen` |
| `-u` | UDP сокеты | Аргумент: флаг `-u` |
| `-l` | только listening | Эффект: только сокеты в состоянии LISTEN |
| `-p` | показать процесс | В выводе: PID и имя процесса (часто нужен `sudo`) |
| `-n` | числовой формат | Эффект: адреса и порты числом, без обратного DNS |

```bash
ss -tulpen
```

```bash
ss -ltnp
```

## Ресурсы и диагностика

### df

занятость файловых систем. **[Подробное описание →](df.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-h` | human-readable размеры | В выводе: колонки Use% и размеры вроде `20G`, `512M` |
| `-i` | usage inode | В выводе: колонка `IUse%` — заполненность inode |

```bash
df -h
```

```bash
df -i
```

### du

размер директорий/файлов. **[Подробное описание →](du.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-s` | summary (без деталей) | Вывод: одна строка с суммарным размером пути |
| `-h` | human-readable | В выводе размер как `1.2G` |
| `--max-depth` | глубина расчета | Аргумент: `--max-depth=1` — только прямые дети каталога |

```bash
du -sh /var/log/*
```

```bash
du -h --max-depth=1 /var/lib
```

### free

использование RAM и swap. **[Подробное описание →](free.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-h` | human-readable | В выводе: `Mi`/`Gi` в строках Mem/Swap |
| `-m` | вывод в мегабайтах | В выводе: числа в мегабайтах (без суффиксов `-h`) |

```bash
free -h
```

### dmesg

сообщения ядра (I/O, OOM, драйверы). **[Подробное описание →](dmesg.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-T` | человеческое время | Вывод: время в читаемом виде вместо секунд с boot |
| `-l` | фильтр уровня | Аргумент: `-l err,warn` — только сообщения выбранных уровней |

```bash
dmesg | tail -n 100
```

```bash
dmesg -T -l err,warn
```

## Безопасность (Debian/Ubuntu)

### sudo

выполнить команду с повышенными правами. **[Подробное описание →](sudo.md)**

| Параметр | Что делает | Пояснение |
| --- | --- | --- |
| `-l` | список разрешённых команд | Вывод (пример): строки `User ... may run ...` или ограничения по путям |
| `-i` | login-оболочка (часто root) | Эффект: сессия ближе к полноценному входу, чем у одной команды |
| `-u пользователь` | от имени другого пользователя | Аргумент: `www-data`, `postgres` и т.д. |
| `-E` | сохранить окружение | Только если sudoers разрешает; иначе часть переменных сбросится |
| `-n` | без запроса пароля | При необходимости пароля — ошибка (удобно в скриптах с жёсткой политикой) |

```bash
sudo systemctl restart nginx
```

```bash
sudo -l
```

### id, sshd -T

контекст пользователя и итоговая SSH-политика. Список правил **sudo**: `sudo -l` — см. [sudo](sudo.md).

| Команда/ключ | Что делает | Пояснение |
| --- | --- | --- |
| `id` | uid/gid/group list | Вывод (пример): `uid=1000(lenar) gid=1000(lenar) groups=...` |
| `sshd -T` | итоговые SSH-настройки | Вывод (пример): `permitrootlogin no`, `passwordauthentication no` |

```bash
id
```

```bash
sudo sshd -T | rg "permitrootlogin|passwordauthentication"
```

### apt, ufw, nft
обновления и firewall.

| Команда | Что делает | Пояснение |
| --- | --- | --- |
| `apt list --upgradable` | доступные обновления | Вывод: список пакетов с пометкой `[upgradable from: ...]` |
| `ufw status verbose` | статус правил UFW | Вывод (пример): `Status: active` и перечень правил |
| `nft list ruleset` | полный ruleset nftables | Вывод: дамп таблиц/цепочек/правил в синтаксисе nft |

```bash
apt list --upgradable
```

```bash
ufw status verbose
```

```bash
nft list ruleset
```
