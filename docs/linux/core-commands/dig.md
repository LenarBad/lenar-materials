# dig

## Назначение

Выполнить **DNS-запрос** из командной строки: получить A/AAAA, NS, MX, отладить делегирование и TTL. Стандарт для проверки резолвинга, когда `ping` по имени ведёт себя непредсказуемо.

## TLDR

Краткий ответ (часто только данные записи).

```bash
dig example.com +short
```

Обычный подробный ответ (вопрос, ответ, авторитет, доп. секции).

```bash
dig example.com
```

Запрос к указанному DNS-серверу.

```bash
dig @1.1.1.1 example.com
```

Только IPv6-адреса.

```bash
dig example.com AAAA +short
```

Обратный резолвинг по IP.

```bash
dig -x 93.184.216.34 +short
```

Трассировка пути резолвинга (отладка делегирования).

```bash
dig example.com +trace
```

## Синтаксис

```text
dig [@сервер] имя [тип] [опции]
```

Тип записи по умолчанию часто **A**; для почты указывают **MX**, для имён серверов — **NS**.

## Поведение и параметры

`dig` отправляет запрос к резолверу: либо к указанному в **`@`**, либо к серверам из `/etc/resolv.conf` (или systemd-resolved, в зависимости от ОС).

| Параметр | Что делает |
| --- | --- |
| `+short` | Сократить вывод до ответа |
| `+trace` | Итеративная трассировка от корневых серверов |
| `-x адрес` | Обратный запрос для PTR |
| `+tcp` | Использовать TCP вместо UDP |
| `+norecurse` | Рекурсия выключена (диагностика) |

Код **`SERVFAIL`** или пустой ответ — повод проверить делегирование, DNSSEC и доступность авторитетных серверов.

## Примеры

Сравнить ответ публичного резолвера и прямой запрос к NS зоны (если известен).

```bash
dig example.com +short
dig @ns1.example.net example.com +short
```
