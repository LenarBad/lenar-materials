# df

## Назначение

Показать **занятость файловых систем**: сколько места занято и свободно на каждом смонтированном разделе, а также (с ключом) **заполненность inode**. Первая команда при ошибках записи и сообщениях «No space left on device».

## TLDR

Человекочитаемые размеры и процент использования.

```bash
df -h
```

Проверить inode (когда место «есть», но файлы не создаются).

```bash
df -i
```

Только определённая файловая система или точка монтирования.

```bash
df -h /var
```

Тип файловой системы в выводе.

```bash
df -hT
```

Синхронизировать данные перед показом (актуальнее, медленнее).

```bash
df -h --sync
```

## Синтаксис

```text
df [опции] [путь ...]
```

Если указан **путь**, показывается файловая система, на которой он лежит.

## Поведение и параметры

`df` смотрит на **уровень раздела**, а не на сумму файлов в каталоге (для этого — **`du`**). Расхождения возможны из‑за снимков, открытых удалённых файлов и квот.

| Параметр | Что делает |
| --- | --- |
| `-h`, `--human-readable` | Размеры в КиБ/МиБ/ГиБ |
| `-i` | Использование inode |
| `-T` | Колонка с типом ФС |
| `-x тип` | Исключить тип ФС (например `tmpfs`) |
| `--sync` | Сбросить буферы перед чтением статистики |

Колонка **`Use%`** около **100%** — сигнал срочно чистить логи/кэш или расширять том.

## Примеры

Быстро найти переполненные разделы.

```bash
df -h | awk 'NR==1 || $5+0 > 80 {print}'
```
