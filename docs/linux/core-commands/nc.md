# nc

## Назначение

**Netcat** — швейцарский нож для TCP/UDP: проверить, открыт ли порт, передать данные вручную, поднять простой listener для отладки. Реализации отличаются (**OpenBSD netcat**, **GNU**, **Ncat** из nmap) — сверяйте ключи с `man nc` на своей системе.

## TLDR

Проверить TCP-порт на хосте (режим без обмена данными).

```bash
nc -zv example.com 443
```

Таймаут ожидания при проверке.

```bash
nc -zvw3 10.0.1.20 5432
```

UDP-порт (диагностика зависит от протокола).

```bash
nc -zvu 10.0.1.1 53
```

Простой listener на порту (для теста; осторожно с firewall).

```bash
nc -l 8080
```

Передать файл по TCP на другой хост (пример идеи; настройте приёмник отдельно).

```bash
nc -N host 9000 < backup.bin
```

## Синтаксис

```text
nc [опции] [хост [порт]]
```

Часть флагов записывается **слитно** (`-zvw3`) или раздельно — зависит от сборки.

## Поведение и параметры

**`-z`** в типичном OpenBSD netcat включает режим «только проверить порт» без полноценной сессии. **`-v`** даёт поясняющий вывод; для скриптов смотрите код выхода.

| Параметр | Что делает (типично) |
| --- | --- |
| `-z` | Скан/проверка: не отправлять лишних данных |
| `-v` | Подробный вывод |
| `-w SEC` | Таймаут |
| `-l` | Режим прослушивания |
| `-u` | UDP вместо TCP |
| `-k` | (где есть) принимать несколько соединений на listener |

Имя **`nc`** может указывать на **`ncat`** или **`netcat`** — справка и набор ключей различаются.

## Примеры

Убедиться, что локальный сервис слушает порт.

```bash
nc -zv 127.0.0.1 6379
```
