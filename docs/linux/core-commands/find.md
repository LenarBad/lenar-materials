# find

## Назначение

**Обойти дерево каталогов** начиная с указанных мест и вывести (или обработать) только те файлы и каталоги, которые подходят под заданные условия: по имени, типу, давности изменения и т.д.

## TLDR

Обычные файлы с шаблоном имени в `/etc`.

```bash
find /etc -type f -name "*.conf"
```

Файлы старше семи суток по дате изменения.

```bash
find /var/log -type f -mtime +7
```

Только объекты на первом уровне внутри каталога (без глубже).

```bash
find /home/USER -maxdepth 1 -mindepth 1
```

Только каталоги с заданным именем.

```bash
find . -type d -name "node_modules"
```

Крупные файлы (GNU find; синтаксис `-size` см. `man find`).

```bash
find /var -type f -size +100M
```

Файлы, принадлежащие указанному пользователю.

```bash
find /var/lib -type f -user www-data
```

Пустые каталоги (удобно чистить «пустые оболочки»).

```bash
find . -type d -empty
```

Печать с подробностями в стиле `ls -dils`.

```bash
find /tmp -maxdepth 1 -ls
```

Передать найденные пути в команду пакетно (один запуск).

```bash
find . -name "*.log" -exec grep -l "ERROR" {} +
```

## Синтаксис

```text
find [с чего начать ...] [условия и действия]
```

Сначала указывают **каталоги отправления** (можно несколько). Дальше идёт цепочка **проверок** (например «тип — файл», «имя по шаблону»), при необходимости **связки** «и / или / отрицание», и в конце — **что сделать** с каждым подошедшим путём. Если действие не указать, обычно просто печатают путь.

На Linux чаще всего это **GNU find**; на других системах набор ключей может отличаться — смотрите `man find` на целевой машине.

## Поведение и параметры

Команда рекурсивно спускается вниз и для каждого объекта решает, проходит ли он все проверки. Сложные условия задают скобками `\(` `\)` и операторами `-and`, `-or`, `!` — порядок важен.

### Частые проверки

| Пример | Смысл |
| --- | --- |
| `-type f` | Только обычные файлы |
| `-type d` | Только каталоги |
| `-type l` | Только символические ссылки |
| `-name 'шаблон'` | Имя подходит под шаблон (`*`, `?`) |
| `-iname 'шаблон'` | То же, без учёта регистра |
| `-path шаблон` | Совпадает весь путь |
| `-mtime +7` | Изменялся раньше чем 7 суток назад |
| `-mtime -1` | Изменялся за последние сутки |
| `-maxdepth n` | Не уходить глубже n уровней от старта |
| `-mindepth n` | Пропустить верхние уровни |
| `-user`, `-group` | Владелец или группа |
| `-perm` | Права доступа (правила см. в `man find`) |

### Что сделать с найденным

| Пример | Смысл |
| --- | --- |
| `-print` | Напечатать путь (часто это действие по умолчанию) |
| `-ls` | Показать строку в стиле списка `ls` |
| `-delete` | Удалить (**сначала** прогоните ту же команду без `-delete` и проверьте список) |
| `-exec команда {} \;` | Запустить команду **отдельно для каждого** пути; `{}` — место под путь |
| `-exec команда {} +` | Собрать **много** путей в один запуск команды (часто быстрее) |
| `-ok команда {} \;` | Как `-exec`, но спрашивает подтверждение перед каждым запуском |

## Примеры

Все файлы `.log` в дереве проекта.

```bash
find ~/project -type f -name "*.log"
```

Каталоги с именем `build` ниже текущего каталога.

```bash
find . -type d -name build
```

Крупные файлы (больше 100 МиБ) в `/var` — синтаксис `-size` зависит от версии `find`, см. `man find`.

```bash
find /var -type f -size +100M
```

Поиск строки во всех найденных файлах за один проход.

```bash
find . -type f -name "*.py" -exec grep -H "import os" {} +
```

Перед удалением: сначала только список путей.

```bash
find /tmp -name "app-*.log" -mtime +30
```

После проверки — удаление найденного.

```bash
find /tmp -name "app-*.log" -mtime +30 -delete
```

Обход очень больших деревьев может занять время — смысленнее начинать с узкого каталога и при необходимости ограничивать глубину.
