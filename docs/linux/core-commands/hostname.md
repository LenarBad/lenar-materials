# hostname

## Назначение

Показать **сетевое имя машины** (то, под которым хост обычно известен в DNS и в логах). Часто используют вместе с `whoami` и `pwd` в «разведке» перед изменениями.

## TLDR

Короткое сетевое имя машины.

```bash
hostname
```

Явно запросить короткое имя.

```bash
hostname -s
```

Полное имя с доменом, если резолвинг и настройки позволяют.

```bash
hostname -f
```

Все IP-адреса хоста, кроме loopback (удобно для быстрой сверки).

```bash
hostname -I
```

Один IP (поведение зависит от версии; см. `man hostname`).

```bash
hostname -i
```

Имя в переменную для скрипта или лога.

```bash
h="$(hostname)"
```

Сводка и политика имён на системах с systemd.

```bash
hostnamectl status
```

## Синтаксис

```text
hostname [опции] [имя]
```

Без аргумента **имя** команда только **показывает** имя. С правами администратора и аргументом **имя** некоторые сборки позволяют **задать** имя до следующей перезагрузки — для постоянной настройки на современных дистрибутивах чаще смотрят `hostnamectl` и конфигурацию сети.

## Поведение и параметры

**Без ключей** обычно выводится короткое имя — то, что вы видите в приглашении оболочки или в заголовке SSH-сессии.

| Параметр | Что делает |
| --- | --- |
| `-f`, `--fqdn` | Попытаться показать полное имя с доменом (если резолвинг и настройки это позволяют) |
| `-I`, `--all-ip-addresses` | Список IP всех адресов хоста, кроме loopback |
| `-i`, `--ip-address` | Один IP (поведение зависит от версии; при сомнениях смотрите `man hostname` на своей системе) |
| `-s`, `--short` | Короткое имя (явно) |

Имя в выводе **может не совпадать** с тем, как хост виден извне: внешнее имя задаёт DNS и облачная панель, локально — `hostname`, cloud-init, NetworkManager. Для согласованной политики имён смотрите также **«статус» хоста** в вашем дистрибутиве (`hostnamectl status` там, где есть systemd).

## Примеры

Зафиксировать имя в переменную для скрипта.

```bash
h="$(hostname)"
echo "Запуск на $h"
```

Сравнить короткое имя и вариант с доменом.

```bash
hostname
hostname -f
```
