# Linux Basics

## Цель
Дать рабочую базу по Linux: как устроена система, как ориентироваться в терминале и как безопасно выполнять ежедневные операции.

## Что выбирать в типовых развилках
| Сценарий | Команда | Когда использовать |
| --- | --- | --- |
| Понять где вы находитесь | `pwd`, `whoami`, `hostname` | pre-flight перед изменениями |
| Быстро посмотреть содержимое | `ls -lah` | обзор директории с правами |
| Понять что делает команда | `<cmd> --help`, `man <cmd>` | перед применением незнакомой команды |
| Проверить результат команды | `echo $?` | контроль успеха/ошибки |
| Записать вывод в файл | `cmd > out.txt` / `cmd >> out.txt` | логирование результата |
| Передать вывод другой команде | `cmd1 | cmd2` | фильтрация/поиск по потоку |

## Shell safety quick notes
| Тема | Практика |
| --- | --- |
| Quoting | использовать `"..."`, если в аргументах возможны пробелы/спецсимволы |
| Wildcards | сначала preview через `ls pattern*`, потом действие |
| Redirects | `>` перезаписывает файл, `>>` дописывает |
| Exit code | `0` - успех, `!=0` - ошибка/особый статус |
| `sudo` | применять точечно к конкретной команде, не ко всей сессии без необходимости |

## Примеры

!!! note "Кто вы и где находитесь"

    Текущий пользователь.

    ```bash
    whoami
    ```

    Текущая директория.

    ```bash
    pwd
    ```

    Имя хоста.

    ```bash
    hostname
    ```

!!! note "Базовая навигация и обзор"

    Список с подробностями.

    ```bash
    ls -la
    ```

    Переход в каталог логов.

    ```bash
    cd /var/log
    ```

!!! note "Справка по команде"

    Справочная страница в пейджере.

    ```bash
    man ls
    ```

    Краткая справка в терминале.

    ```bash
    ls --help
    ```

!!! note "Коды возврата"

    Сначала команда, затем проверка кода выхода.

    ```bash
    ls /tmp
    ```

    ```bash
    echo $?
    ```

!!! note "Перенаправление и конвейер"

    Записать список файлов в файл.

    ```bash
    ls -lah /etc > /tmp/etc-list.txt
    ```

    Прочитать файл и отфильтровать строки.

    ```bash
    cat /tmp/etc-list.txt | rg ssh
    ```

## Частые ошибки
- выполнять команды с `sudo` без необходимости;
- путать относительные и абсолютные пути;
- не проверять текущую директорию перед удалением/перемещением;
- запускать команды с wildcard без предварительного preview;
- игнорировать код возврата после критичных операций.
