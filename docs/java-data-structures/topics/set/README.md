# `Set`

`Set` — коллекция **без дубликатов**: два элемента не могут быть «равны» в смысле `equals` (для хеш-реализаций дополнительно нужен согласованный `hashCode`).

## Контракт `Set` (главные методы)

- `add(E)` — добавить элемент, вернёт `false`, если эквивалентный уже есть.
- `contains(Object)` — проверка наличия.
- `remove(Object)` — удаление элемента.
- `size()`, `isEmpty()`, `iterator()` — размер, пустота, обход.
- `addAll(...)`, `removeAll(...)`, `retainAll(...)` — массовые операции множеств.

## Основные реализации и стоимость операций

| Реализация | Когда уместна | Преимущества | Ограничения | `add` | `contains` | `remove` | Порядок обхода |
|-----------|----------------|--------------|-------------|-------|------------|----------|----------------|
| `HashSet` | Нужна максимальная средняя скорость без требований к порядку | Быстрые операции в среднем | Нет гарантии порядка | среднее `O(1)` | среднее `O(1)` | среднее `O(1)` | не гарантирован |
| `LinkedHashSet` | Нужен predictable-порядок + быстрые операции | Сохраняет порядок вставки | Чуть больше накладных расходов, чем у `HashSet` | среднее `O(1)` | среднее `O(1)` | среднее `O(1)` | порядок вставки |
| `TreeSet` | Нужна сортировка и диапазонные операции | Отсортированные элементы, `lower/higher`, `subSet` | Медленнее хеш-структур на базовых операциях | `O(log n)` | `O(log n)` | `O(log n)` | сортировка |

## Обход

- Базовый вариант: `for (E e : set)` или `set.forEach(...)`.
- Если важен порядок, используй `LinkedHashSet` (вставка) или `TreeSet` (сортировка).
- У `HashSet` порядок итерации не часть контракта и не должен влиять на логику.
- Для удаления при обходе — `Iterator.remove()` или `removeIf(...)`.

## Примеры

<div class="lenar-tip-grid" markdown="block">

!!! note "`HashSet`, `LinkedHashSet`, `TreeSet` в одном фрагменте"

    Здесь видно разницу в назначении: хеш-набор, набор с порядком вставки и отсортированный набор.
    По сложности вставки: `HashSet`/`LinkedHashSet` — в среднем `O(1)`, `TreeSet` — `O(log n)`.

    ```java
    import java.util.HashSet;
    import java.util.LinkedHashSet;
    import java.util.Set;
    import java.util.TreeSet;

    Set<String> h = new HashSet<>();
    h.add("b");
    h.add("a");

    Set<String> l = new LinkedHashSet<>();
    l.add("b");
    l.add("a"); // обход чаще сохраняет порядок вставки

    Set<Integer> t = new TreeSet<>();
    t.add(10);
    t.add(2);
    ```

</div>

## Частые ошибки

- Класть в `HashSet`/`LinkedHashSet` объекты с **некорректным или изменяемым** `hashCode`/`equals` — дубликаты или «потерянные» элементы.
- Ожидать от `HashSet` стабильного порядка итерации — его нет в контракте.
