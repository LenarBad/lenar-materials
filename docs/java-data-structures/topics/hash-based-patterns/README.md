# Хеш-ориентированные паттерны

## Цель

Собрать типовые алгоритмические приёмы на базе `HashMap` и `HashSet`: подсчёт частот, дедупликация, быстрые проверки принадлежности.

Когда применять:

- нужна проверка "видели ли элемент раньше";
- нужен frequency map (подсчёт вхождений);
- нужно группировать элементы по ключу.

## Суть подхода

- Храним дополнительное состояние в хеш-структуре: "видели ли", "сколько раз видели", "к какой группе относится".
- На каждом шаге делаем локальное обновление `Map`/`Set`, избегая вложенных циклов.
- Типичный выигрыш: переход от `O(n^2)` к `O(n)` в среднем за счёт `O(1)` операций хеш-таблицы.
- Корректность зависит от контрактов `equals/hashCode` для ключей и элементов.

## Примеры

<div class="lenar-tip-grid" markdown="block">

!!! note "Подсчёт частот через `HashMap`"

    Увеличиваем счётчик каждого слова через `merge`.
    Если ключа ещё нет, в `Map` записывается `1`. Если ключ уже есть, вызывается `Integer::sum(oldValue, 1)`.
    Сложность: один проход `O(n)` по словам, обновление на шаге — в среднем `O(1)`.

    ```java
    import java.util.HashMap;
    import java.util.Map;

    String[] words = {"a", "b", "a"};
    Map<String, Integer> freq = new HashMap<>();
    for (String w : words) {
        freq.merge(w, 1, Integer::sum);
    }
    ```

!!! note "Проверка дублей через `HashSet`"

    `Set.add(x)` возвращает `true`, если элемент добавлен впервые, и `false`, если такой элемент уже есть.
    Поэтому условие `!seen.add(x)` срабатывает именно на дубликате.
    Сложность: один проход `O(n)`, `add` на шаге — в среднем `O(1)`.

    ```java
    import java.util.HashSet;
    import java.util.Set;

    int[] nums = {1, 2, 3, 2};
    Set<Integer> seen = new HashSet<>();
    boolean hasDuplicate = false;
    for (int x : nums) {
        if (!seen.add(x)) {
            hasDuplicate = true;
            break;
        }
    }
    ```

## Частые ошибки

- Использовать изменяемые объекты как ключи, если их поля участвуют в `equals`/`hashCode`.
- Полагаться на порядок обхода `HashMap`/`HashSet`.
- Писать логику через `containsKey` + `put` вместо более компактного `merge`/`compute` без необходимости.
