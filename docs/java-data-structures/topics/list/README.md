# `List`

`List` — упорядоченная коллекция с доступом по индексу; один и тот же элемент может встречаться несколько раз.

## Контракт `List` (главные методы)

- `add(E)` / `add(int, E)` — добавить элемент (в конец или по индексу).
- `get(int)`, `set(int, E)` — чтение и замена по индексу.
- `remove(int)` / `remove(Object)` — удаление по индексу или значению.
- `contains(Object)`, `indexOf(Object)` — поиск в списке.
- `size()`, `isEmpty()`, `iterator()` — базовые операции и обход.

## Основные реализации и стоимость операций

| Реализация | Когда уместна | Преимущества | Ограничения | `get(i)` | `add` в конец | вставка/удаление в середине | `contains` |
|-----------|----------------|--------------|-------------|----------|---------------|-----------------------------|------------|
| `ArrayList` | Универсальный выбор по умолчанию | Плотное хранение, быстрый доступ по индексу | Сдвиги элементов при вставке/удалении не в конце | `O(1)` | амортизированно `O(1)` | `O(n)` | `O(n)` |
| `LinkedList` | Частые операции на концах и работа через `Deque`/итератор | Добавление/удаление на концах `O(1)` | Медленный доступ по индексу, больше накладные расходы памяти | `O(n)` | `O(1)` | `O(n)`* | `O(n)` |

\* Если уже есть позиция итератора на нужном узле, локальная вставка/удаление выполняется за `O(1)`.

Для прикладного кода без особых требований обычно начинают с `ArrayList`.

## Обход

- Для чтения: `for (E e : list)` или `list.forEach(...)`.
- Для `ArrayList` индексный цикл уместен: `for (int i = 0; i < list.size(); i++)`.
- Для `LinkedList` лучше итератор/for-each, а не частый `get(i)`.
- Удаление во время обхода: `Iterator.remove()` или `removeIf(...)`.

## Примеры

<div class="lenar-tip-grid" markdown="block">

!!! note "Сравнение `ArrayList` и `LinkedList`"

    `ArrayList` удобен как базовый список, `LinkedList` может быть полезен как `Deque`.
    В этом фрагменте: `a.add(...)` для `ArrayList` — амортизированно `O(1)`, `b.addFirst(...)` для `LinkedList` — `O(1)`, `b.addAll(a)` — `O(n)` по числу добавляемых элементов.

    ```java
    import java.util.ArrayList;
    import java.util.LinkedList;
    import java.util.List;

    List<Integer> a = new ArrayList<>();
    a.add(1);
    a.add(2);

    List<Integer> b = new LinkedList<>();
    b.addFirst(0); // метод Deque у LinkedList
    b.addAll(a);
    ```

</div>

## Частые ошибки

- Часто вызывать `list.get(i)` в цикле по большому **`LinkedList`** — получается квадратичная стоимость; для обхода используй **итератор** или расширенный `for`.
- Использовать `subList` без понимания: представление привязано к исходному `List`; изменения в одном месте отражаются в другом, пока живы оба представления.
