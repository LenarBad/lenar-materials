# Алгоритмическая сложность и выбор структуры

## Цель

Дать практичную модель выбора коллекций по стоимости операций, а не по привычке. Для большинства задач сначала оценивают, какие операции доминируют (`get`, `contains`, `add`, `remove`, итерация), и только потом выбирают `List`/`Set`/`Map` и реализацию.

Ориентиры:

- `ArrayList`: быстрый индексный доступ, дорогие вставки/удаления в середине.
- `HashMap`/`HashSet`: быстрые операции в среднем, порядок не гарантирован.
- `TreeMap`/`TreeSet`: медленнее, но дают сортировку и range-операции.
- `ArrayDeque`: хороший выбор по умолчанию для очереди/стека.

## Суть подхода

- Сначала определяем профиль задачи: какие операции выполняются чаще всего (`get`, `contains`, `insert`, `remove`, обход).
- Выбираем структуру под доминирующие операции, а не "самую знакомую".
- Считаем не только Big-O, но и практические факторы: память, константы, предсказуемость порядка.
- Проверяем гипотезу измерением на реальных данных, если участок кода критичен по производительности.

## Примеры

<div class="lenar-tip-grid" markdown="block">

!!! note "Серия операций и амортизированная стоимость"

    Мини-задача: есть поток чисел, нужно накопить их в списке, а затем посчитать частоту по группам (`x % 100`).
    Пример показывает, почему для такого сценария обычно берут `ArrayList` + `HashMap`.
    Итого по фрагменту: построение списка `O(n)`, заполнение `Map` ещё `O(n)` в среднем, суммарно `O(n)` по времени и `O(n)` по памяти.

    ```java
    import java.util.ArrayList;
    import java.util.HashMap;
    import java.util.List;
    import java.util.Map;

    List<Integer> nums = new ArrayList<>(); // 1) Накапливаем входные данные
    for (int i = 0; i < 100_000; i++) {
        nums.add(i); // амортизированно O(1)
    }

    Map<Integer, Integer> freq = new HashMap<>(); // 2) Считаем частоты по корзинам 0..99
    for (int x : nums) {
        freq.merge(x % 100, 1, Integer::sum); // в среднем O(1) на шаг
    }
    // 3) В freq лежит результат: ключ — корзина, значение — сколько чисел в ней оказалось
    ```

</div>

## Частые ошибки

- Сравнивать структуры только по Big-O и игнорировать константы, память и читаемость.
- Брать `LinkedList` "на всякий случай", хотя сценарий требует в основном `get(i)` и итерацию.
- Ожидать от `HashMap`/`HashSet` стабильного порядка обхода.
