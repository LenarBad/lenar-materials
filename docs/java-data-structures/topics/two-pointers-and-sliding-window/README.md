# Два указателя и sliding window

## Цель

Показать два базовых паттерна для задач на массивы и строки, которые хорошо сочетаются с коллекциями Java:

- **два указателя** (`left/right`) для упорядоченных данных;
- **sliding window** для подотрезков и подстрок.

Обычно используют `ArrayList`/массивы + `HashMap` для частот и `Deque` для некоторых оконных задач.

## Суть паттернов

- **Два указателя**: держим два индекса (`left`, `right`) и двигаем их по правилу задачи, чтобы не перебирать все пары/подотрезки. Часто даёт линейное решение вместо квадратичного.
- **Sliding window**: рассматриваем текущее «окно» (`left..right`), расширяем его вправо и при нарушении условия сужаем слева. Подходит для задач про подстроки/подмассивы с ограничением (без повторов, сумма не больше `k`, частоты и т.п.).
- Ключевая идея у обоих: границы двигаются монотонно вперёд, поэтому каждый элемент обычно обрабатывается ограниченное число раз.

## Примеры

!!! note "Sliding window без повторяющихся символов"

    Задача: найти длину самой длинной подстроки без повторяющихся символов.
    `left` двигается только вперёд, а `Map` хранит последнюю позицию каждого символа.
    Сложность: `O(n)` по времени (каждый индекс обрабатывается константное число раз), `O(k)` по памяти для множества символов в окне.

    ```java
    import java.util.HashMap;
    import java.util.Map;

    String s = "abcaef";
    Map<Character, Integer> last = new HashMap<>();
    int left = 0;
    int best = 0; // лучший (максимальный) размер окна без повторов

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (last.containsKey(c)) {
            // Если символ уже был в текущем окне, сдвигаем левую границу правее прошлого вхождения.
            left = Math.max(left, last.get(c) + 1);
        }
        // Запоминаем последнюю позицию символа и обновляем лучший ответ.
        last.put(c, right);
        best = Math.max(best, right - left + 1);
    }
    ```

</div>

## Частые ошибки

- Неправильно двигать `left`, из-за чего окно "откатывается назад".
- Забывать обновлять структуру состояния окна (`Map`/`Set`) при сдвиге границы.
- Применять этот паттерн там, где проще решить через сортировку или префиксные суммы.
