# `Map`

`Map` — сопоставление **уникального ключа** значению. Один ключ в один момент времени соответствует не более чем одному значению (в `HashMap` можно положить `null` как ключ или значение — помни о последствиях для читаемости).

## Контракт `Map` (главные методы)

- `put(K, V)`, `get(K)`, `remove(K)` — базовые операции по ключу.
- `containsKey(K)`, `containsValue(V)` — проверки наличия.
- `putIfAbsent`, `compute`, `merge`, `getOrDefault` — атомарные/компактные паттерны обновления.
- `keySet()`, `values()`, `entrySet()` — представления для обхода.
- `size()`, `isEmpty()`, `clear()` — базовое управление.

## Основные реализации и стоимость операций

| Реализация | Когда уместна | Преимущества | Ограничения | `put` | `get` | `remove` | Порядок обхода |
|-----------|----------------|--------------|-------------|-------|-------|----------|----------------|
| `HashMap` | Выбор по умолчанию без требований к порядку | Быстрые операции в среднем | Порядок не гарантирован | среднее `O(1)` | среднее `O(1)` | среднее `O(1)` | не гарантирован |
| `LinkedHashMap` | Нужен стабильный порядок или access-order для LRU | Предсказуемый обход, удобен для кэшей | Небольшие накладные расходы относительно `HashMap` | среднее `O(1)` | среднее `O(1)` | среднее `O(1)` | вставка / доступ |
| `TreeMap` | Нужна сортировка ключей и диапазонные запросы | Отсортированные ключи, `first/last`, `subMap` | Базовые операции медленнее, чем у хеш-таблиц | `O(log n)` | `O(log n)` | `O(log n)` | сортировка по ключу |

## Полезные операции

- `getOrDefault`, `putIfAbsent`, `compute`, `merge` — уменьшают ручные `if` вокруг отсутствующих ключей (используй осознанно, чтобы не усложнить чтение).

## Обход

- Самый практичный вариант: обход через `entrySet()` (есть и ключ, и значение сразу).
- `keySet()` уместен, если нужны только ключи; `values()` — только значения.
- Порядок обхода зависит от реализации: у `HashMap` не гарантирован, у `LinkedHashMap` предсказуем, у `TreeMap` отсортирован.
- Удалять элементы во время обхода безопасно через итератор по `entrySet()`.

!!! note "Базовые операции `Map` и безопасное обновление"

    Пример показывает обычный `put`, `putIfAbsent` и `merge` для инкремента значения.
    Для `HashMap` эти операции в среднем `O(1)` на вызов.

    ```java
    import java.util.HashMap;
    import java.util.Map;

    Map<String, Integer> scores = new HashMap<>();
    scores.put("Ann", 10);
    scores.putIfAbsent("Bob", 0);
    scores.merge("Ann", 1, Integer::sum);
    ```

## Частые ошибки

- **Мутировать ключ** после вставки в `HashMap`/`LinkedHashMap`, если от изменённого поля зависит `hashCode` — таблица перестаёт находить запись.
- Использовать `get` без проверки на `null` значения, когда ключ отсутствует — учитывай `containsKey` или `getOrDefault`.
