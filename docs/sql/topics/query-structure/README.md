# Структура запроса на выборку

## Цель
Понимать, из каких **логических шагов** собирается типичный запрос на чтение: что выполняется первым и что попадает в результат. Эта тема про **одну строку таблицы как кандидата в ответ** до группировки; свёртка в группы и агрегаты — в [Aggregation](../aggregation/README.md).

## Декларативность
SQL-запрос описывает **какой** результат нужен, а не обязательно порядок физических операций в СУБД. Оптимизатор может переставить соединения и способы доступа к данным, но **смысл** результата задаётся стандартом и вашим текстом запроса.

## Логический порядок частей (ориентир)
В учебниках часто приводят такую последовательность (точные имена и опции зависят от диалекта):

1. **`FROM`** — из каких таблиц (и как соединённых) берутся строки-«кандидаты».
2. **`WHERE`** — какие из этих строк **отбрасываются** до группировки.
3. **`GROUP BY` / `HAVING`** — если нужна агрегация (см. [Aggregation](../aggregation/README.md)).
4. **`SELECT`** — какие **столбцы и выражения** попадают в результат (проекция).
5. **`ORDER BY`** — порядок строк **после** вычисления проекции.
6. **`LIMIT` / `OFFSET`** (или `FETCH`) — сколько строк вернуть клиенту.

На практике важно помнить: **`WHERE` не видит** имена столбцов, введённые только в списке `SELECT` как алиасы — об этом в [Имена, алиасы и CTE](../identifiers-and-cte/README.md).

## `SELECT`: проекция
Список после `SELECT` задаёт, **какие поля и выражения** будут в каждой строке ответа: оригинальные столбцы, вычисления, литералы, вызовы функций. Имя столбца в результате можно задать через `AS` (алиас).

## `FROM`: источник строк
Указывает одну таблицу, представление (`VIEW`) или подзапрос. Несколько таблиц и условия их связи — через запятую (старый стиль) или явные `JOIN`; подробно — [Joins](../joins/README.md).

## `WHERE`: фильтр по строке
Условие вычисляется для каждой строки (или для строки после соединений). Строки, для которых результат — «ложь» или *unknown* (см. [«NULL» и уникальность](../null-and-uniqueness/README.md)), в дальнейшую обработку не передаются.

Типовые предикаты — сравнения, `IN`, `BETWEEN`, `LIKE`, логические `AND`/`OR`. Готовые короткие шаблоны — в [шаблонах запросов](../../cheatsheets/query-patterns.md).

## `ORDER BY` и ограничение числа строк
- **`ORDER BY`** задаёт сортировку результата. Без неё порядок строк **не гарантирован** (если только ваша СУБД не документирует иное для простого случая).
- **`LIMIT` / `OFFSET`** (или эквивалент) отрезает «хвост» после сортировки. Глубокий `OFFSET` на больших смещениях часто дорог; альтернатива — пагинация по ключу.

!!! note "Минимальный запрос: откуда и что взять"

    ```sql
    SELECT id, name, created_at
    FROM users
    WHERE deleted_at IS NULL;
    ```

!!! note "Сортировка и ограничение выборки"

    ```sql
    SELECT id, name
    FROM users
    WHERE country = 'RU'
    ORDER BY created_at DESC
    LIMIT 100;
    ```
