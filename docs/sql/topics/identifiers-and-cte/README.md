# Идентификаторы, алиасы и обобщённые табличные выражения (`WITH`)

## Цель
Писать запросы, которые **переносятся между средами** (регистр имён, кавычки), избегать ловушки с **алиасами в `WHERE`**, и применять **`WITH` (CTE)**, когда так проще прочитать и сопровождать логику.

## Имена таблиц и столбцов
В стандарте незаключённые идентификаторы часто приводятся к определённому регистру (зависит от СУБД). Если имена приходят из внешних систем или содержат пробелы и спецсимволы — используйте **кавычки** по правилам вашей СУБД (`"Имя"`, `` `имя` ``, и т.д.) и будьте последовательны: смешение закавыченных и обычных имён легко даёт ошибку «такого столбца нет».

## Алиасы (`AS`) в списке `SELECT`
Алиас задаёт имя столбца **в результате** запроса. На этапе вычисления **`WHERE`** эти имена **ещё не существуют** для текущего уровня `SELECT`: фильтр применяется к строкам источника до формирования проекции с алиасами. Поэтому нельзя писать `WHERE gross > 100`, если `gross` только что введён в том же `SELECT` — нужно повторить выражение или использовать подзапрос / CTE.

<div class="lenar-tip-grid" markdown="block">

!!! note "Фильтр по выражению и по алиасу через подзапрос"

    Сначала — то же выражение, что и в проекции:

    ```sql
    SELECT user_id, amount * 1.2 AS gross
    FROM payments
    WHERE amount * 1.2 > 100;
    ```

    Вариант с вложенным запросом, где имя `gross` уже есть во внутреннем результате:

    ```sql
    SELECT user_id, gross
    FROM (
      SELECT user_id, amount * 1.2 AS gross
      FROM payments
    ) s
    WHERE gross > 100;
    ```

</div>

## CTE: `WITH` …
**CTE** расшифровывается как **Common Table Expression** (в русской терминологии SQL — то же «обобщённое табличное выражение»). Это **именованный подзапрос**, на который ссылаются дальше в том же запросе:

```sql
WITH имя AS (
  SELECT ...
)
SELECT ...
FROM имя ...
```

### Когда имеет смысл
- Длинная цепочка условий или несколько шагов «сначала отфильтровать множество, потом соединить».
- Одна и та же промежуточная выборка используется **дважды** — без CTE её пришлось бы дублировать.
- Читаемость важнее нескольких строк вложенности.

Оптимизатор часто **инлайнит** CTE как обычный подзапрос, но не стоит считать CTE автоматической «материализацией» без проверки плана.

<div class="lenar-tip-grid" markdown="block">

!!! note "Два шага: множество пользователей и факты по заказам"

    ```sql
    WITH активные AS (
      SELECT user_id
      FROM users
      WHERE deleted_at IS NULL
    )
    SELECT a.user_id, o.amount
    FROM активные AS a
    JOIN orders AS o ON o.user_id = a.user_id;
    ```

</div>
