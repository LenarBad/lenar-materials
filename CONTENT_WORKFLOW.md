# Content Workflow

Этот документ фиксирует как добавлять и обновлять материалы в проекте.

## 1) Что считается источником истины
- Канонический контент живет в `topics/*`.
- Форматы потребления (`learning`, `cheatsheets`, `recipes`) дают адаптированную подачу и ссылки на темы.

## 2) Сценарий: добавить новую тему
1. Создать `topics/<topic>/README.md`.
2. Добавить блок `Где использовать` со ссылками на релевантные mode-файлы.
3. Обновить минимум один mode-файл (`learning/`, `cheatsheets/`, `recipes/`) ссылкой на новую тему.
4. Проверить навигацию в `mkdocs.yml`.

## 3) Сценарий: добавить контент в существующую тему
1. Сначала обновить тему в `topics/*`.
2. Затем синхронизировать соответствующие mode-файлы:
   - `learning` — объяснение,
   - `cheatsheets` — короткая выжимка,
   - `recipes` — пошаговые сценарии.
3. Вместо копипаста — ставить ссылки на canonical-часть.

## 4) Мини-шаблон страницы
Рекомендуемые секции:
- `## Цель`
- `## Когда использовать`
- `## Пошаговый подход`
- `## Примеры`
- `## Частые ошибки`
- `## Проверка результата`
- `## Связанные материалы`

## 5) Публикация
- Редактируем только Markdown.
- Локальная проверка: `mkdocs serve`.
- Сборка: `mkdocs build`.
- Публикация: GitHub Actions в GitHub Pages.
